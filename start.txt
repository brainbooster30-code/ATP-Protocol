Specifica Tecnica (Revisione Architetturale)

1. Abstract (invariato)
L'Agent Transport Protocol (ATP) definisce un livello applicativo sicuro per lo scambio di task e informazioni tra agenti autonomi. Il protocollo combina identità crittografiche basate su Merkle-Claim Cards (MCC), trasporto autenticato tramite QUIC + TLS 1.3 con raw public keys (RFC 7250), e un sistema di revoca distribuita basato su Cuckoo Filters sincronizzati via gossip epidemic.

2. Modifiche e Correzioni Apportate
2.1 Root Store e Bootstrap: Introdotto il Root Store Chain
Problema: Singolo punto di fallimento (bootstrap OOB).

Correzione: Il Root Store viene ora distribuito come una catena di manifest firmati, aggiornabile in-band tramite ROOT_STORE_UPDATE.

Il genesis manifest è distribuito OOB con hash pinnato.

Ogni nuovo manifest (manifest_version incrementale) DEVE essere firmato da almeno quorum_threshold autorità non revocate.

La verifica è ricorsiva: ogni nuovo manifest deve puntare al precedente (prev_manifest_id), formando una blockchain leggera.

Se un agente riceve un manifest con versione superiore, lo verifica e lo adotta; se la verifica fallisce, lo scarta.

Ciò elimina la dipendenza da un singolo bootstrap e permette aggiornamenti sicuri dell'autorità.

Nuovo campo in root_store_manifest:

root_store_manifest = {
  "manifest_version": uint,
  "manifest_id": bstr .size 16,
  "prev_manifest_id": bstr .size 16,   ; null UUID per genesis
  "timestamp": uint,
  "quorum_threshold": uint,
  "authorities": [+ authority_entry],
  "signatures": [+ manifest_signature],
  ? "gossip_params": { ... }
}
2.2 Revoca e Fail-Closed: Politica di Degradazione Gracevole
Problema: Revoca troppo rigida (fail-closed) in caso di incertezza.

Correzione: Introdotto un stato di revoca graduale con tre livelli:

CONFIRMED: lo snapshot ha meno di grace_period_s secondi. Il filtro è affidabile.

STALE: lo snapshot ha tra grace_period_s e 2 * grace_period_s. La revoca è ancora accettata ma viene loggato un warning.

UNCERTAIN: lo snapshot è più vecchio di 2 * grace_period_s o non verificabile. In questo stato:

Se il task ha capabilities critiche → fail-closed (ERR_REVOKED).

Se il task non ha capabilities critiche → fail-open (task accettato, log di REVOCATION_UNCERTAIN).

Nuova logica:

python
if age_s < grace_period_s:
    status = CONFIRMED
elif age_s < 2 * grace_period_s:
    status = STALE
else:
    status = UNCERTAIN

if status == UNCERTAIN:
    if critical_capabilities_present:
        return ERR_REVOKED
    else:
        accept_and_log_uncertain()
Inoltre, il gossip è ora self-throttling: ogni peer accetta al massimo 1 nuovo snapshot ogni gossip_interval_s (default 5s), e lo snapshot deve essere incrementale (filter_id crescente), non più grande di max_snapshot_size (default 1 MiB).

@dataclass
class MCCLeaf:
    key: str
    value: bytes
    salt: bytes  # 16 bytes

    def compute_leaf_hash(self) -> bytes:
        # L_i = BLAKE3(0x00 || salt || uint16_BE(len(key)) || key_bytes || uint32_BE(len(value)) || value)
        key_bytes = self.key.encode('utf-8')
        val_bytes = self.value
        data = b'\x00' + self.salt + struct.pack('>H', len(key_bytes)) + key_bytes + struct.pack('>I', len(val_bytes)) + val_bytes
        return blake3_hash(data)

    @staticmethod
    def _build_merkle_root(leaf_hashes: List[bytes]) -> bytes:
        # Build complete binary tree per RFC 6962 §2.1
        n = len(leaf_hashes)
        if n == 0:
            raise ValueError("no leaves")
        if n == 1:
            return leaf_hashes[0]
        # Pad to power of two by duplicating last leaf
        import math
        target = 1 << (n - 1).bit_length()
        if n < target:
            # Duplicate last leaf
            leaf_hashes = leaf_hashes + [leaf_hashes[-1]] * (target - n)
        # Build tree
        level = leaf_hashes
        while len(level) > 1:
            next_level = []
            for i in range(0, len(level), 2):
                left = level[i]
                right = level[i+1] if i+1 < len(level) else left
                node = blake3_hash(b'\x01' + left + right)
                next_level.append(node)
            level = next_level
        return level[0]

        self.identity = identity
        self.is_server = is_server
        self.state = "INIT"
        self.peer_mcc: Optional[MCC] = None

    # ----- Handshake (over a stream) -----
    async def perform_handshake(self, reader: asyncio.StreamReader, writer: asyncio.StreamWriter):
        if self.is_server:
            # Server: wait for VERSION_PROPOSE
            frame = await self._read_frame(reader)
            if frame["frame_type"] != FRAME_TYPES["VERSION_PROPOSE"]:
                raise Exception("Expected VERSION_PROPOSE")
            # Respond with VERSION_ACK
            ack_frame = {
                "header": build_header(FRAME_TYPES["VERSION_ACK"]),
                "atp_version": ATP_VERSION,
                "max_batch_bytes": MAX_BATCH_BYTES,
                "clock_skew_ms": CLOCK_SKEW_MS,
                "anti_replay_ttl_ms": ANTI_REPLAY_TTL_MS,
                "rate_limit_rps": RATE_LIMIT_RPS,
            }
            await self._send_frame(writer, ack_frame)

            # Wait for MCC_BIND_REQUEST
            frame = await self._read_frame(reader)
            if frame["frame_type"] != FRAME_TYPES["MCC_BIND_REQUEST"]:
                raise Exception("Expected MCC_BIND_REQUEST")
            mcc_data = frame["mcc"]
            peer_mcc = MCC.from_cbor(cbor2.dumps(mcc_data, canonical=True))
            # Verify peer MCC using authority pk from Root Store (hardcoded for demo)
            authority_pk = get_default_authority().pk_bytes
            ok, err = peer_mcc.verify(authority_pk, tls_pk=None)  # we skip TLS key match for demo
            if not ok:
                raise Exception(f"Peer MCC verification failed: {err}")
            self.peer_mcc = peer_mcc
            self.nonce_i = frame["nonce"]
            # Build response: our MCC, nonce_r, signature over nonce_i
            nonce_r = os.urandom(16)
            self.nonce_r = nonce_r
            sig = ed25519_sign(self.identity.ed25519_sk, self.nonce_i + b"atp-bind-response")
            response_frame = {
                "header": build_header(FRAME_TYPES["MCC_BIND_RESPONSE"]),
                "mcc": cbor2.loads(self.identity.mcc.to_cbor()),
                "nonce": nonce_r,
                "nonce_sig": sig
            }
            await self._send_frame(writer, response_frame)

            # Wait for MCC_BIND_CONFIRM
            frame = await self._read_frame(reader)
            if frame["frame_type"] != FRAME_TYPES["MCC_BIND_CONFIRM"]:
                raise Exception("Expected MCC_BIND_CONFIRM")
            sig = frame["nonce_sig"]
            # Verify signature over nonce_r
            if not ed25519_verify(self.peer_mcc.leaves[0].value, self.nonce_r + b"atp-bind-confirm", sig):  # agent_sign_pk
                raise Exception("Bind confirm signature invalid")
            self.bound = True
            logger.info("Handshake complete (server)")
        else:
            # Client: send VERSION_PROPOSE
            propose_frame = {
                "header": build_header(FRAME_TYPES["VERSION_PROPOSE"]),
                "atp_versions": [ATP_VERSION],
                "max_batch_bytes": MAX_BATCH_BYTES,
                "clock_skew_ms": CLOCK_SKEW_MS,
                "anti_replay_ttl_ms": ANTI_REPLAY_TTL_MS,
                "rate_limit_rps": RATE_LIMIT_RPS,
            }
            await self._send_frame(writer, propose_frame)

            # Wait for VERSION_ACK
            frame = await self._read_frame(reader)
            if frame["frame_type"] != FRAME_TYPES["VERSION_ACK"]:
                raise Exception("Expected VERSION_ACK")
            # Send MCC_BIND_REQUEST
            nonce_i = os.urandom(16)
            self.nonce_i = nonce_i
            request_frame = {
                "header": build_header(FRAME_TYPES["MCC_BIND_REQUEST"]),
                "mcc": cbor2.loads(self.identity.mcc.to_cbor()),
                "nonce": nonce_i
            }
            await self._send_frame(writer, request_frame)

            # Wait for MCC_BIND_RESPONSE
            frame = await self._read_frame(reader)
            if frame["frame_type"] != FRAME_TYPES["MCC_BIND_RESPONSE"]:
                raise Exception("Expected MCC_BIND_RESPONSE")
            peer_mcc_data = frame["mcc"]
            peer_mcc = MCC.from_cbor(cbor2.dumps(peer_mcc_data, canonical=True))
            authority_pk = get_default_authority().pk_bytes
            ok, err = peer_mcc.verify(authority_pk, tls_pk=None)
            if not ok:
                raise Exception(f"Peer MCC verification failed: {err}")
            self.peer_mcc = peer_mcc
            self.nonce_r = frame["nonce"]
            sig = frame["nonce_sig"]
            # Verify signature over our nonce_i
            if not ed25519_verify(self.peer_mcc.leaves[0].value, self.nonce_i + b"atp-bind-response", sig):
                raise Exception("Peer signature on nonce_i invalid")
            # Send confirm
            confirm_sig = ed25519_sign(self.identity.ed25519_sk, self.nonce_r + b"atp-bind-confirm")
            confirm_frame = {
                "header": build_header(FRAME_TYPES["MCC_BIND_CONFIRM"]),
                "nonce_sig": confirm_sig
            }
            await self._send_frame(writer, confirm_frame)
            self.bound = True
            logger.info("Handshake complete (client)")

    # ----- Task exchange -----
    async def send_task(self, writer: asyncio.StreamWriter, task_type: str, payload: bytes, deadline_ms: int = 5000) -> Dict[str, Any]:
        if not self.bound:
            raise Exception("Not bound")
        task_id = uuid.uuid4().bytes
        frame = {
            "header": build_header(FRAME_TYPES["TASK_REQUEST"], task_id),
            "task_type": task_type,
            "task_payload": payload,
            "deadline_ms": deadline_ms,
            "metadata": {"agent_id": self.identity.agent_id}
        }
        await self._send_frame(writer, frame)
        # Wait for TASK_ACK
        ack = await self._read_frame(writer)
        if ack["frame_type"] != FRAME_TYPES["TASK_ACK"]:
            raise Exception("Expected TASK_ACK")
        # Wait for TASK_RESPONSE
        response = await self._read_frame(writer)
        if response["frame_type"] != FRAME_TYPES["TASK_RESPONSE"]:
            if response["frame_type"] == FRAME_TYPES["TASK_ERROR"]:
                raise Exception(f"Task error: {response.get('error_message')}")
            raise Exception("Expected TASK_RESPONSE")
        return response

    async def handle_task(self, reader: asyncio.StreamReader, writer: asyncio.StreamWriter):
        # Called by server to process incoming tasks
        while True:
            frame = await self._read_frame(reader)
            if frame is None:
                break
            if frame["frame_type"] == FRAME_TYPES["TASK_REQUEST"]:
                # Send TASK_ACK
                ack_frame = {
                    "header": build_header(FRAME_TYPES["TASK_ACK"]),
                    "received_at": int(time.time() * 1000)
                }
                await self._send_frame(writer, ack_frame)
                # Process task
                task_type = frame["task_type"]
                task_payload = frame["task_payload"]
                logger.info(f"Received task: {task_type}")
                # Dispatch to callback if registered
                result = None
                if task_type in self.task_callbacks:
                    result = await self.task_callbacks[task_type](task_payload, frame["header"]["task_id"])
                else:
                    result = {"error": "unsupported task type"}
                # Send response
                response_frame = {
                    "header": build_header(FRAME_TYPES["TASK_RESPONSE"]),
                    "status": 0 if result else 1,
                    "result_payload": cbor2.dumps(result) if result else b''
                }
                await self._send_frame(writer, response_frame)
                # Close stream after response (in real ATP, server sends FIN, but we just close)
                writer.close()
                await writer.wait_closed()
                break
            else:
                logger.warning(f"Unexpected frame type: {frame['frame_type']}")

    # ----- LLM integration -----
    @staticmethod
    async def call_deepseek(prompt: str) -> str:
        async with aiohttp.ClientSession() as session:
            headers = {
                "Authorization": f"Bearer {DEEPSEEK_API_KEY}",
                "Content-Type": "application/json"
            }
            data = {
                "model": DEEPSEEK_MODEL,
                "messages": [{"role": "user", "content": prompt}],
                "stream": False
            }
            async with session.post(DEEPSEEK_API_URL, headers=headers, json=data) as resp:
                if resp.status != 200:
                    text = await resp.text()
                    raise Exception(f"DeepSeek API error {resp.status}: {text}")
                result = await resp.json()
                return result["choices"][0]["message"]["content"]

    # ----- I/O helpers -----
    async def _read_frame(self, reader: asyncio.StreamReader) -> Optional[Dict[str, Any]]:
        # Read 4-byte length prefix
        len_data = await reader.readexactly(4)
        if not len_data:
            return None
        length = struct.unpack('>I', len_data)[0]
        data = await reader.readexactly(length)
        return decode_frame(data)

    async def _send_frame(self, writer: asyncio.StreamWriter, frame: Dict[str, Any]):
        data = encode_frame(frame)
        writer.write(struct.pack('>I', len(data)) + data)
        await writer.drain()

# ----- Helper to create identity -----
def create_agent_identity(agent_id: str, authority) -> AgentIdentity:
    x_sk, x_pk = generate_x25519_keypair()
    e_sk, e_pk = generate_ed25519_keypair()
    # Build MCC leaves
    leaves = [
        MCCLeaf(key="agent_pk", value=x_pk, salt=os.urandom(16)),
        MCCLeaf(key="agent_sign_pk", value=e_pk, salt=os.urandom(16)),
        MCCLeaf(key="expiry_date", value=str(int(time.time()) + 86400).encode(), salt=os.urandom(16)),
        MCCLeaf(key="authority_id", value=authority.authority_id.encode(), salt=os.urandom(16)),
        MCCLeaf(key="mcc_version", value=b'\x01', salt=os.urandom(16)),
        MCCLeaf(key="serial_number", value=uuid.uuid4().bytes, salt=os.urandom(16)),
    ]
    mcc = authority.sign_mcc(leaves, expiry_seconds=86400)
    return AgentIdentity(agent_id, x_sk, x_pk, e_sk, e_pk, mcc)

async def agent_connection_handler(reader, writer, agent: ATPAgent):
    try:
        # Perform handshake (server side)
        await agent.perform_handshake(reader, writer)
        # After handshake, handle incoming tasks
        await agent.handle_task(reader, writer)
    except Exception as e:
        logger.error(f"Connection error: {e}")
    finally:
        writer.close()
        await writer.wait_closed()

async def run_client():
    authority = get_default_authority()
    identity = create_agent_identity("agent-client", authority)
    agent = ATPAgent(identity, is_server=False)

    # TLS context (skip verification)
    context = ssl.create_default_context(ssl.Purpose.SERVER_AUTH)
    context.check_hostname = False
    context.verify_mode = ssl.CERT_NONE

    reader, writer = await asyncio.open_connection('127.0.0.1', 8443, ssl=context)
    try:
        await agent.perform_handshake(reader, writer)
        # Send a task
        prompt = "Explain the concept of autonomous agents in AI in three sentences."

        print("Task response:")
        print(result.get("response", "No response"))

        logger.error(f"Client error: {e}")
    finally:
        writer.close()
        await writer.wait_closed()

Il protocollo combina identità crittografiche basate su **Merkle-Claim Cards (MCC)**, trasporto autenticato tramite **QUIC + TLS 1.3** con raw public keys (RFC 7250), e un sistema di revoca distribuita basato su **Cuckoo Filters** sincronizzati via gossip epidemic.

**Caratteristiche principali di ATP v1.6.1**

- **Identità crittografiche auto-contenute (MCC):** alberi Merkle con domain‑separation, firma Ed25519 su commitment CBOR canonicalizzato, binding esplicito tra identità e chiave TLS. **Rimosso il campo `leaf_hash` dal wire format** per eliminare ambiguità implementative.

Abstract

   The Agent Transport Protocol (ATP) defines a secure application-level
   protocol for the exchange of tasks and information between autonomous
   AI agents.  ATP combines cryptographic identities based on Merkle-
   Claim Cards (MCC), mutually authenticated transport over QUIC + TLS
   1.3 with raw public keys (RFC 7250), and a distributed revocation
   subsystem based on Cuckoo Filters synchronised via gossip epidemic
   propagation.

   backwards-compatible with all conformant ATP 1.5.x and 1.6.x

   improvements: a Root Store chain for secure updates, a graduated
   revocation policy (CONFIRMED/STALE/UNCERTAIN), mandatory key
   separation, removal of the ambiguous leaf_hash field, a clock-skew
   fallback mechanism, and delegation support.

import os
import uuid
import time
import struct
from typing import Dict, Any, Optional, List, Tuple
from dataclasses import dataclass

from cryptography.hazmat.primitives import serialization
from cryptography.hazmat.primitives.asymmetric import ed25519, x25519
import blake3
import cbor2

from config import *

# ---------- Cryptographic utilities ----------

def generate_x25519_keypair() -> Tuple[bytes, bytes]:
    private = x25519.X25519PrivateKey.generate()
    public = private.public_key()
    return private.private_bytes(
        encoding=serialization.Encoding.Raw,
        format=serialization.PrivateFormat.Raw,
        encryption_algorithm=serialization.NoEncryption()
    ), public.public_bytes(encoding=serialization.Encoding.Raw, format=serialization.PublicFormat.Raw)

def generate_ed25519_keypair() -> Tuple[bytes, bytes]:
    private = ed25519.Ed25519PrivateKey.generate()
    public = private.public_key()
    return private.private_bytes(
        encoding=serialization.Encoding.Raw,
        format=serialization.PrivateFormat.Raw,
        encryption_algorithm=serialization.NoEncryption()
    ), public.public_bytes(encoding=serialization.Encoding.Raw, format=serialization.PublicFormat.Raw)

def ed25519_sign(sk_bytes: bytes, data: bytes) -> bytes:
    private = ed25519.Ed25519PrivateKey.from_private_bytes(sk_bytes)
    return private.sign(data)

def ed25519_verify(pk_bytes: bytes, data: bytes, sig: bytes) -> bool:
    try:
        public = ed25519.Ed25519PublicKey.from_public_bytes(pk_bytes)
        public.verify(sig, data)
        return True
    except Exception:
        return False

def blake3_hash(data: bytes) -> bytes:
    return blake3.blake3(data).digest(length=32)

# ---------- MCC ----------

@dataclass
class MCCLeaf:
    key: str
    value: bytes
    salt: bytes  # 16 bytes

    def compute_leaf_hash(self) -> bytes:
        # L_i = BLAKE3(0x00 || salt || uint16_BE(len(key)) || key_bytes || uint32_BE(len(value)) || value)
        key_bytes = self.key.encode('utf-8')
        val_bytes = self.value
        data = b'\x00' + self.salt + struct.pack('>H', len(key_bytes)) + key_bytes + struct.pack('>I', len(val_bytes)) + val_bytes
        return blake3_hash(data)

@dataclass
class MCC:
    mcc_version: int
    serial_number: bytes  # UUID raw, 16 bytes
    root_hash: bytes      # 32 bytes
    authority_id: str
    authority_sig: bytes  # 64 bytes
    expiry_date: int      # Unix seconds
    leaves: List[MCCLeaf]
    critical_mask: List[str]

    def to_cbor(self) -> bytes:
        leaves_cbor = []
        for leaf in self.leaves:
            leaves_cbor.append({
                "key": leaf.key,
                "value": leaf.value,
                "salt": leaf.salt
            })
        obj = {
            "mcc_version": self.mcc_version,
            "serial_number": self.serial_number,
            "root_hash": self.root_hash,
            "authority_id": self.authority_id,
            "authority_sig": self.authority_sig,
            "expiry_date": self.expiry_date,
            "leaves": leaves_cbor,
            "critical_mask": self.critical_mask
        }
        return cbor2.dumps(obj, canonical=True)

    @classmethod
    def from_cbor(cls, data: bytes):
        obj = cbor2.loads(data)
        leaves = [MCCLeaf(key=leaf["key"], value=leaf["value"], salt=leaf["salt"]) for leaf in obj["leaves"]]
        return cls(
            mcc_version=obj["mcc_version"],
            serial_number=obj["serial_number"],
            root_hash=obj["root_hash"],
            authority_id=obj["authority_id"],
            authority_sig=obj["authority_sig"],
            expiry_date=obj["expiry_date"],
            leaves=leaves,
            critical_mask=obj["critical_mask"]
        )

    def verify(self, authority_pk: bytes, tls_pk: Optional[bytes] = None, check_revocation: bool = False) -> Tuple[bool, Optional[str]]:
        # Step 1: version
        if self.mcc_version != 1:
            return False, "ERR_ATP_VERSION_UNSUPPORTED"
        # Step 2: expiry
        now = int(time.time())
        if self.expiry_date <= now:
            return False, "ERR_REVOKED"
        # Step 3: critical_mask
        leaf_keys = {leaf.key for leaf in self.leaves}
        for claim in self.critical_mask:
            if claim not in leaf_keys:
                return False, "ERR_MISSING_CRITICAL_CLAIM"
        # Step 4: recompute root
        leaf_hashes = [leaf.compute_leaf_hash() for leaf in sorted(self.leaves, key=lambda l: l.key)]
        if len(leaf_hashes) == 0:
            return False, "ERR_INVALID_ROOT"
        if len(leaf_hashes) > 1:
            computed_root = self._build_merkle_root(leaf_hashes)
        else:
            computed_root = leaf_hashes[0]
        if computed_root != self.root_hash:
            return False, "ERR_INVALID_ROOT"
        # Step 5-6: verify signature
        commitment_obj = {
            "root_hash": self.root_hash,
            "expiry_date": self.expiry_date,
            "mcc_version": self.mcc_version,
            "authority_id": self.authority_id,
            "serial_number": self.serial_number
        }
        commitment_cbor = cbor2.dumps(commitment_obj, canonical=True)
        if not ed25519_verify(authority_pk, commitment_cbor, self.authority_sig):
            return False, "ERR_BAD_SIGNATURE"
        # Step 7: TLS public key match (optional)
        if tls_pk is not None:
            agent_pk_leaf = next((leaf for leaf in self.leaves if leaf.key == "agent_pk"), None)
            if agent_pk_leaf is None or agent_pk_leaf.value != tls_pk:
                return False, "ERR_IDENTITY_MISMATCH"
        # Step 8: revocation (skip)
        if check_revocation:
            pass
        return True, None

    @staticmethod
    def _build_merkle_root(leaf_hashes: List[bytes]) -> bytes:
        n = len(leaf_hashes)
        if n == 0:
            raise ValueError("no leaves")
        if n == 1:
            return leaf_hashes[0]
        import math
        target = 1 << (n - 1).bit_length()
        if n < target:
            leaf_hashes = leaf_hashes + [leaf_hashes[-1]] * (target - n)
        level = leaf_hashes
        while len(level) > 1:
            next_level = []
            for i in range(0, len(level), 2):
                left = level[i]
                right = level[i+1] if i+1 < len(level) else left
                node = blake3_hash(b'\x01' + left + right)
                next_level.append(node)
            level = next_level
        return level[0]

    @classmethod
    def create(cls, authority_sk: bytes, authority_id: str, leaves: List[MCCLeaf], expiry_seconds: int = 3600) -> 'MCC':
        leaf_hashes = [leaf.compute_leaf_hash() for leaf in sorted(leaves, key=lambda l: l.key)]
        root_hash = cls._build_merkle_root(leaf_hashes)
        serial = uuid.uuid4().bytes
        expiry = int(time.time()) + expiry_seconds
        mcc_version = 1
        commitment_obj = {
            "root_hash": root_hash,
            "expiry_date": expiry,
            "mcc_version": mcc_version,
            "authority_id": authority_id,
            "serial_number": serial
        }
        commitment_cbor = cbor2.dumps(commitment_obj, canonical=True)
        sig = ed25519_sign(authority_sk, commitment_cbor)
        critical_mask = ["agent_pk", "agent_sign_pk", "expiry_date", "authority_id", "mcc_version", "serial_number"]
        return cls(
            mcc_version=mcc_version,
            serial_number=serial,
            root_hash=root_hash,
            authority_id=authority_id,
            authority_sig=sig,
            expiry_date=expiry,
            leaves=leaves,
            critical_mask=critical_mask
        )

def build_header(frame_type: int, task_id: Optional[bytes] = None) -> Dict[str, Any]:
    return {
        "frame_type": frame_type,
        "frame_id": uuid.uuid4().bytes,
        "task_id": task_id or (b'\x00' * 16),
        "timestamp": int(time.time() * 1000),
        "atp_version": ATP_VERSION
    }

    # ----- Handshake -----
    async def perform_handshake(self, reader: asyncio.StreamReader, writer: asyncio.StreamWriter):
        if self.is_server:
            # Server: wait for VERSION_PROPOSE
            frame = await self._read_frame(reader)
            if frame["frame_type"] != FRAME_TYPES["VERSION_PROPOSE"]:
                raise Exception("Expected VERSION_PROPOSE")
            ack_frame = {
                "header": build_header(FRAME_TYPES["VERSION_ACK"]),
                "atp_version": ATP_VERSION,
                "max_batch_bytes": MAX_BATCH_BYTES,
                "clock_skew_ms": CLOCK_SKEW_MS,
                "anti_replay_ttl_ms": ANTI_REPLAY_TTL_MS,
                "rate_limit_rps": RATE_LIMIT_RPS,
            }
            await self._send_frame(writer, ack_frame)

            frame = await self._read_frame(reader)
            if frame["frame_type"] != FRAME_TYPES["MCC_BIND_REQUEST"]:
                raise Exception("Expected MCC_BIND_REQUEST")
            peer_mcc = MCC.from_cbor(cbor2.dumps(frame["mcc"], canonical=True))
            authority_pk = get_default_authority().pk_bytes
            ok, err = peer_mcc.verify(authority_pk, tls_pk=None)
            if not ok:
                raise Exception(f"Peer MCC verification failed: {err}")
            self.peer_mcc = peer_mcc
            self.nonce_i = frame["nonce"]
            nonce_r = os.urandom(16)
            self.nonce_r = nonce_r
            sig = ed25519_sign(self.identity.ed25519_sk, self.nonce_i + b"atp-bind-response")
            response_frame = {
                "header": build_header(FRAME_TYPES["MCC_BIND_RESPONSE"]),
                "mcc": cbor2.loads(self.identity.mcc.to_cbor()),
                "nonce": nonce_r,
                "nonce_sig": sig
            }
            await self._send_frame(writer, response_frame)

            frame = await self._read_frame(reader)
            if frame["frame_type"] != FRAME_TYPES["MCC_BIND_CONFIRM"]:
                raise Exception("Expected MCC_BIND_CONFIRM")
            sig = frame["nonce_sig"]
            agent_sign_pk = next((leaf.value for leaf in self.peer_mcc.leaves if leaf.key == "agent_sign_pk"), None)
            if agent_sign_pk is None or not ed25519_verify(agent_sign_pk, self.nonce_r + b"atp-bind-confirm", sig):
                raise Exception("Bind confirm signature invalid")
            self.bound = True
            logger.info("Handshake complete (server)")
        else:
            # Client
            propose_frame = {
                "header": build_header(FRAME_TYPES["VERSION_PROPOSE"]),
                "atp_versions": [ATP_VERSION],
                "max_batch_bytes": MAX_BATCH_BYTES,
                "clock_skew_ms": CLOCK_SKEW_MS,
                "anti_replay_ttl_ms": ANTI_REPLAY_TTL_MS,
                "rate_limit_rps": RATE_LIMIT_RPS,
            }
            await self._send_frame(writer, propose_frame)

            frame = await self._read_frame(reader)
            if frame["frame_type"] != FRAME_TYPES["VERSION_ACK"]:
                raise Exception("Expected VERSION_ACK")

            nonce_i = os.urandom(16)
            self.nonce_i = nonce_i
            request_frame = {
                "header": build_header(FRAME_TYPES["MCC_BIND_REQUEST"]),
                "mcc": cbor2.loads(self.identity.mcc.to_cbor()),
                "nonce": nonce_i
            }
            await self._send_frame(writer, request_frame)

            frame = await self._read_frame(reader)
            if frame["frame_type"] != FRAME_TYPES["MCC_BIND_RESPONSE"]:
                raise Exception("Expected MCC_BIND_RESPONSE")
            peer_mcc = MCC.from_cbor(cbor2.dumps(frame["mcc"], canonical=True))
            authority_pk = get_default_authority().pk_bytes
            ok, err = peer_mcc.verify(authority_pk, tls_pk=None)
            if not ok:
                raise Exception(f"Peer MCC verification failed: {err}")
            self.peer_mcc = peer_mcc
            self.nonce_r = frame["nonce"]
            sig = frame["nonce_sig"]
            agent_sign_pk = next((leaf.value for leaf in self.peer_mcc.leaves if leaf.key == "agent_sign_pk"), None)
            if agent_sign_pk is None or not ed25519_verify(agent_sign_pk, self.nonce_i + b"atp-bind-response", sig):
                raise Exception("Peer signature on nonce_i invalid")
            confirm_sig = ed25519_sign(self.identity.ed25519_sk, self.nonce_r + b"atp-bind-confirm")
            confirm_frame = {
                "header": build_header(FRAME_TYPES["MCC_BIND_CONFIRM"]),
                "nonce_sig": confirm_sig
            }
            await self._send_frame(writer, confirm_frame)
            self.bound = True
            logger.info("Handshake complete (client)")

    # ----- Task exchange -----
    async def send_task(self, writer: asyncio.StreamWriter, task_type: str, payload: bytes, deadline_ms: int = 5000) -> Dict[str, Any]:
        if not self.bound:
            raise Exception("Not bound")
        task_id = uuid.uuid4().bytes
        frame = {
            "header": build_header(FRAME_TYPES["TASK_REQUEST"], task_id),
            "task_type": task_type,
            "task_payload": payload,
            "deadline_ms": deadline_ms,
            "metadata": {"agent_id": self.identity.agent_id}
        }
        await self._send_frame(writer, frame)
        ack = await self._read_frame(reader)
        if ack["frame_type"] != FRAME_TYPES["TASK_ACK"]:
            raise Exception("Expected TASK_ACK")
        response = await self._read_frame(reader)
        if response["frame_type"] != FRAME_TYPES["TASK_RESPONSE"]:
            if response["frame_type"] == FRAME_TYPES["TASK_ERROR"]:
                raise Exception(f"Task error: {response.get('error_message')}")
            raise Exception("Expected TASK_RESPONSE")
        return response

    async def handle_task(self, reader: asyncio.StreamReader, writer: asyncio.StreamWriter):
        while True:
            frame = await self._read_frame(reader)
            if frame is None:
                break
            if frame["frame_type"] == FRAME_TYPES["TASK_REQUEST"]:
                ack_frame = {
                    "header": build_header(FRAME_TYPES["TASK_ACK"]),
                    "received_at": int(time.time() * 1000)
                }
                await self._send_frame(writer, ack_frame)
                task_type = frame["task_type"]
                task_payload = frame["task_payload"]
                logger.info(f"Received task: {task_type}")
                result = None
                if task_type in self.task_callbacks:
                    result = await self.task_callbacks[task_type](task_payload, frame["header"]["task_id"])
                else:
                    result = {"error": "unsupported task type"}
                response_frame = {
                    "header": build_header(FRAME_TYPES["TASK_RESPONSE"]),
                    "status": 0 if result else 1,
                    "result_payload": cbor2.dumps(result) if result else b''
                }
                await self._send_frame(writer, response_frame)
                writer.close()
                await writer.wait_closed()
                break
            else:
                logger.warning(f"Unexpected frame type: {frame['frame_type']}")

    # ----- LLM integration -----
    @staticmethod
    async def call_deepseek(prompt: str) -> str:
        async with aiohttp.ClientSession() as session:
            headers = {
                "Authorization": f"Bearer {DEEPSEEK_API_KEY}",
                "Content-Type": "application/json"
            }
            data = {
                "model": DEEPSEEK_MODEL,
                "messages": [{"role": "user", "content": prompt}],
                "stream": False
            }
            async with session.post(DEEPSEEK_API_URL, headers=headers, json=data) as resp:
                if resp.status != 200:
                    text = await resp.text()
                    raise Exception(f"DeepSeek API error {resp.status}: {text}")
                result = await resp.json()
                return result["choices"][0]["message"]["content"]

    # ----- I/O helpers -----
    async def _read_frame(self, reader: asyncio.StreamReader) -> Optional[Dict[str, Any]]:
        try:
            len_data = await reader.readexactly(4)
        except asyncio.IncompleteReadError:
            return None
        length = struct.unpack('>I', len_data)[0]
        data = await reader.readexactly(length)
        return decode_frame(data)

    async def _send_frame(self, writer: asyncio.StreamWriter, frame: Dict[str, Any]):
        data = encode_frame(frame)
        writer.write(struct.pack('>I', len(data)) + data)
        await writer.drain()

def create_agent_identity(agent_id: str, authority) -> AgentIdentity:
    x_sk, x_pk = generate_x25519_keypair()
    e_sk, e_pk = generate_ed25519_keypair()
    leaves = [

        MCCLeaf(key="agent_sign_pk", value=e_pk, salt=os.urandom(16)),

    return AgentIdentity(agent_id, x_sk, x_pk, e_sk, e_pk, mcc)

async def run_server():
    authority = get_default_authority()

    agent.task_callbacks["deepseek.generate"] = deepseek_callback

    # Generate self‑signed TLS certificate
    key = rsa.generate_private_key(public_exponent=65537, key_size=2048)
    subject = issuer = x509.Name([x509.NameAttribute(NameOID.COMMON_NAME, "localhost")])
    cert = (
        x509.CertificateBuilder()
        .subject_name(subject)
        .issuer_name(issuer)
        .public_key(key.public_key())
        .serial_number(x509.random_serial_number())
        .not_valid_before(datetime.utcnow())
        .not_valid_after(datetime.utcnow() + timedelta(days=1))
        .add_extension(x509.SubjectAlternativeName([x509.DNSName("localhost")]), critical=False)
        .sign(key, hashes.SHA256())
    )
    with tempfile.NamedTemporaryFile(delete=False, suffix=".pem") as keyfile:
        keyfile.write(key.private_bytes(encoding=serialization.Encoding.PEM,
                                         format=serialization.PrivateFormat.PKCS8,
                                         encryption_algorithm=serialization.NoEncryption()))

        certfile.write(cert.public_bytes(serialization.Encoding.PEM))
        cert_path = certfile.name

    context = ssl.create_default_context(ssl.Purpose.CLIENT_AUTH)
    context.load_cert_chain(cert_path, key_path)

    server = await asyncio.start_server(
        lambda r, w: agent_connection_handler(r, w, agent),
        host='127.0.0.1', port=8443, ssl=context
    )

Sei un senior software engineer specializzato in protocolli di rete, crittografia e sistemi distribuiti. Ti viene richiesto di implementare una versione funzionante del protocollo Agent Transport Protocol (ATP) v1.6.1 in Python, seguendo fedelmente la specifica tecnica allegata.

Il protocollo ATP è progettato per permettere a agenti autonomi (basati su LLM) di comunicare in modo sicuro, verificando l'identità tramite Merkle-Claim Cards (MCC), autenticando il trasporto con TLS, e gestendo la revoca distribuita tramite Cuckoo Filters e gossip.

## SPECIFICA TECNICA ATP v1.6.1 (sintetica)

### 1. Identità: Merkle-Claim Cards (MCC)

Ogni agente possiede una MCC, che è un albero Merkle (MAST) costruito con BLAKE3, con encoding CBOR canonicale e firmato da un'autorità.

**Formule crittografiche:**
- Foglia: `L_i = BLAKE3(0x00 || salt_16byte || uint16_BE(len(key)) || key_utf8 || uint32_BE(len(value)) || value)`
- Nodo interno: `Node = BLAKE3(0x01 || Left || Right)`
- Albero completo (padding con ultima foglia se N non è potenza di 2)
- Root hash per N=1: `root_hash = L_0`

**Verifica MCC (8 passi obbligatori):**
1. Verificare `mcc_version` supportata (v1)
2. Verificare `expiry_date > now`
3. Verificare che tutti i campi in `critical_mask` esistano tra le foglie
4. Ricalcolare `leaf_hash` e `root_hash` (ignorando eventuali hash trasmessi)
5. Recuperare `authority_pk` dal Root Store locale
6. Verificare firma Ed25519 su commitment CBOR (5 campi: root_hash, expiry_date, mcc_version, authority_id, serial_number)

### 2. Trasporto e Handshake

**Stack:** QUIC v1 + TLS 1.3 con raw public keys (RFC 7250), ALPN "atp/1".

**Handshake (5 fasi):**

**Fase 1 – QUIC + TLS:** Initiator apre connessione, raw_public_key = agent_pk.

**Fase 2 – Version Negotiation (stream 0):**
- Initiator → VERSION_PROPOSE (0x30): atp_versions, max_batch_bytes, clock_skew_ms, anti_replay_ttl_ms, rate_limit_rps
- Responder → VERSION_ACK (0x31): atp_version scelta, parametri negoziati

**Fase 3 – MCC Exchange & Identity Binding (stream 0):**
- Initiator → MCC_BIND_REQUEST (0x40): MCC + nonce_i (16 byte CSPRNG)
- Responder verifica MCC; risponde con MCC_BIND_RESPONSE (0x41): propria MCC + nonce_r + sig_Ed25519(nonce_i || "atp-bind-response")
- Initiator verifica; risponde con MCC_BIND_CONFIRM (0x42): sig_Ed25519(nonce_r || "atp-bind-confirm")
- Responder verifica → binding completato (bidirezionale)

**Fase 4 – Capability Exchange (opzionale, stream 0):** CAPABILITY_EXCHANGE (0x50) con campo "required".

**Fase 5 – Task Streams:** stream pari (2, 4, 6...) per TASK_REQUEST/RESPONSE.

### 4. Revoca e Root Store (ATP-Full)

**Cuckoo Filter:**
- false positive ≤ 0.1%, bucket_size = 4, fingerprint_bits = 16
- fingerprint = BLAKE3(serial_number || expiry_date_bytes)[0:2] (0x0000 → 0x0001)
- h1 = BLAKE3(0x68 || x) mod bucket_count
- h2 = (h1 XOR (BLAKE3(0x66 || fp(x)) mod bucket_count)) mod bucket_count
- Formato binario CFT1: magic "CFT1" + bucket_count + bucket_size (4) + fingerprint_bits (16) + data

**Gossip:**
- max 1 snapshot / gossip_interval_s per sender
- max_snapshot_size = 1 MiB
- filter_id crescente (incrementale)

**Politica di degradazione:**
- CONFIRMED: age < grace_period_s (300s) → fiducia piena
- STALE: grace_period_s ≤ age < 2*grace_period_s → accettato con warning
- UNCERTAIN: age ≥ 2*grace_period_s → se critical capabilities → fail-closed (ERR_REVOKED), altrimenti fail-open

**Root Store Chain:**
root_store_manifest = {
"manifest_version": uint,
"manifest_id": bstr .size 16,

"timestamp": uint,
"quorum_threshold": uint,
"authorities": [+ authority_entry],
"signatures": [+ manifest_signature],
? "gossip_params": { ... }
}

## SPECIFICA TECNICA ATP v1.6.1 (sintetica ma vincolante)

### 1. Identità: Merkle-Claim Cards (MCC)

Ogni agente possiede una MCC, che è un albero Merkle (MAST) costruito con BLAKE3, con encoding CBOR canonicale e firmato da un'autorità.

**Formule crittografiche:**
- Foglia: `L_i = BLAKE3(0x00 || salt_16byte || uint16_BE(len(key)) || key_utf8 || uint32_BE(len(value)) || value)`
- Nodo interno: `Node = BLAKE3(0x01 || Left || Right)`
- Albero completo (padding con ultima foglia se N non è potenza di 2)
- Root hash per N=1: `root_hash = L_0`

**Verifica MCC (8 passi obbligatori):**
1. Verificare `mcc_version` supportata (v1)
2. Verificare `expiry_date > now`
3. Verificare che tutti i campi in `critical_mask` esistano tra le foglie
4. Ricalcolare `leaf_hash` e `root_hash` (ignorando eventuali hash trasmessi)
5. Recuperare `authority_pk` dal Root Store locale
6. Verificare firma Ed25519 su commitment CBOR (5 campi: root_hash, expiry_date, mcc_version, authority_id, serial_number)
7. Verificare che `agent_pk` corrisponda alla chiave TLS (opzionale in demo)
8. (ATP-Full) Verificare `serial_number` non revocato

**CDDL MCC (NOTA: leaf_hash rimosso):**
mcc = {
"mcc_version": uint,
"serial_number": bstr .size 16,
"root_hash": bstr .size 32,
"authority_id": tstr .size (1..256),
"authority_sig": bstr .size 64,
"expiry_date": uint,
"leaves": [+ mcc_leaf],
"critical_mask": [+ tstr]
}
mcc_leaf = {
"key": tstr .size (1..128),
"value": bstr,
"salt": bstr .size 16
/* leaf_hash NON DEVE essere trasmesso – il receiver ricalcola */
}

text

**Critical mask obbligatoria:** agent_pk, agent_sign_pk, expiry_date, authority_id, mcc_version, serial_number

**Separazione chiavi (ATP-Full):**
- `agent_pk` (X25519, 32 byte) → solo ECDH TLS
- `agent_sign_pk` (Ed25519, 32 byte) → solo firme MCC e proof-of-possession
- `agent_pk != agent_sign_pk` obbligatorio in ATP-Full
- Dual-use deprecato; se usato, flag `"dual_use": true`

### 2. Trasporto e Handshake

**Stack:** TCP + TLS self‑signed (simula QUIC), ALPN non necessario in demo.

**Handshake (5 fasi):**
- **Fase 1 – TLS:** Connessione TCP con TLS (self‑signed).
- **Fase 2 – Version Negotiation (stream 0):**
  - Initiator → VERSION_PROPOSE (0x30): atp_versions, max_batch_bytes, clock_skew_ms, anti_replay_ttl_ms, rate_limit_rps
  - Responder → VERSION_ACK (0x31): atp_version scelta, parametri negoziati
- **Fase 3 – MCC Exchange & Identity Binding:**
  - Initiator → MCC_BIND_REQUEST (0x40): MCC + nonce_i (16 byte CSPRNG)
  - Responder verifica MCC; risponde con MCC_BIND_RESPONSE (0x41): propria MCC + nonce_r + sig_Ed25519(nonce_i || "atp-bind-response")
  - Initiator verifica; risponde con MCC_BIND_CONFIRM (0x42): sig_Ed25519(nonce_r || "atp-bind-confirm")
  - Responder verifica → binding completato (bidirezionale)
- **Fase 4 – Capability Exchange (opzionale, ma implementata):** CAPABILITY_EXCHANGE (0x50) con campo "required".
- **Fase 5 – Task Streams:** stream paralleli (multiplexing simulato con più task in sequenza).

## 1. Abstract

L'**Agent Transport Protocol (ATP)** definisce un livello applicativo sicuro per lo scambio di task e informazioni tra agenti autonomi. Il protocollo combina identità crittografiche basate su **Merkle-Claim Cards (MCC)**, trasporto autenticato tramite **QUIC + TLS 1.3** con raw public keys (RFC 7250), e un sistema di revoca distribuita basato su **Cuckoo Filters** sincronizzati via gossip epidemic.

**Caratteristiche principali di ATP v1.6.1**

- **Identità crittografiche auto-contenute (MCC):** alberi Merkle con domain‑separation, firma Ed25519 su commitment CBOR canonicalizzato, binding esplicito tra identità e chiave TLS. **Rimosso il campo `leaf_hash` dal wire format** per eliminare ambiguità implementative.
- **Trasporto zero‑trust:** autenticazione mutua obbligatoria al binding, proof‑of‑possession bidirezionale, nessuna PKI X.509.
- **Revoca distribuita:** Cuckoo Filters sincronizzati via gossip epidemic (ATP‑Full), con **politica di degradazione graduale** (CONFIRMED / STALE / UNCERTAIN) e fallback OCSP‑like. **Introdotta catena di aggiornamento del Root Store** per eliminare il singolo punto di fallimento del bootstrap.
- **State machine normative:** connessione, control stream e task stream completamente specificati con tutti gli stati e le transizioni.
- **Profili di conformità:** ATP‑Core‑Minimal per ambienti embedded/chiusi; ATP‑Full per sistemi distribuiti ad alta sicurezza. **Separazione delle chiavi obbligatoria** in ATP‑Full; dual‑use deprecato.

---

## 2. Architettura dell'Identità: Merkle‑Claim Cards (MCC)

### 2.1 Struttura dell'Albero Merkle

Ogni foglia è calcolata con la seguente formula:
L_i = BLAKE3(
0x00 || // prefisso dominio foglia (OBBLIGATORIO)
Salt_128bit || // 16 byte, CSPRNG, unico per foglia
uint16_BE(len(Key_bytes)) || // lunghezza chiave, big-endian
Key_bytes || // chiave in UTF-8
uint32_BE(len(Val_bytes)) || // lunghezza valore, big-endian
Val_bytes // valore
)

text

**Ordinamento delle foglie:** le chiavi sono ordinate per confronto byte-per-byte della rappresentazione UTF-8 in ordine lessicografico crescente. In caso di prefisso comune, la chiave più corta precede. L'indice `leaf_index` è 0-based.

**Costruzione dell'albero:**

- L'albero è un albero binario completo (complete binary tree).
- Se N non è una potenza di 2, l'ultimo livello viene completato duplicando l'ultima foglia (RFC 6962 §2.1).
- **Nodi interni:** `Node = BLAKE3(0x01 || Left || Right)`. Il prefisso 0x01 è OBBLIGATORIO per prevenire attacchi di second-preimage.
- **Caso speciale — albero a singola foglia (N = 1):** `root_hash = L₀`. Non si applica alcun hashing aggiuntivo.

### 2.2 Firma dell'Autorità e Procedura di Verifica MCC

L'autorità firma un oggetto di commitment CBOR canonicalizzato (RFC 8949). L'oggetto firmato contiene esattamente 5 campi e NON include il campo "leaves".

**Oggetto firmato (ordine chiavi RFC 8949 §4.2.1):**
{
"root_hash": bstr .size 32, // 9 byte
"expiry_date": uint, // 11 byte
"mcc_version": uint, // 11 byte ("e" < "m")
"authority_id": tstr, // 12 byte
"serial_number": bstr .size 16 // 13 byte
}

authority_sig = Ed25519Sign(authority_sk, CBOR_canonical(commitment))

text

**Procedura normativa di verifica MCC (8 passi, in ordine):**

1. Verificare che `mcc_version` sia supportata → `ERR_ATP_VERSION_UNSUPPORTED`
2. Verificare che `expiry_date > now_ms / 1000` → `ERR_REVOKED`
3. Verificare che tutti i campi in `critical_mask` esistano tra le foglie. Claim in `critical_mask` non riconosciuto → `ERR_MISSING_CRITICAL_CLAIM` (fail-closed)
4. Ricalcolare ogni `leaf_hash` da `key/value/salt` (formula §2.1). Il receiver DEVE ignorare `leaf_hash` trasmesso nel wire format. Ricalcolare `root_hash`; verificare che coincida con `mcc.root_hash` → `ERR_INVALID_ROOT`
5. Recuperare `authority_pk` dal Root Store locale. Se assente o revocata → `ERR_INVALID_ROOT`
6. Verificare `authority_sig = Ed25519Verify(authority_pk, CBOR_canonical(commitment))` → `ERR_BAD_SIGNATURE`
7. Verificare che `agent_pk` nel leaf coincida con la raw public key TLS → `ERR_IDENTITY_MISMATCH`
8. (ATP-Full) Verificare `serial_number` non presente nel Cuckoo Filter locale → `ERR_REVOKED`. In ATP-Core-Minimal: facoltativo.

> **NOTA:** Il campo `leaf_hash` nel wire format è stato **RIMOSSO**. Il receiver DEVE sempre ricalcolare il leaf hash. Un forger che altera key/value/salt viene rilevato al passo 4.

### 2.3 Critical Mask

**Campi obbligatori (sempre in critical_mask):**

| Campo | Tipo e Vincoli |
|-------|----------------|
| `agent_pk` | Chiave X25519, 32 byte |
| `agent_sign_pk` | Chiave Ed25519, 32 byte (vedi §2.5) |
| `expiry_date` | uint64, Unix epoch in secondi |
| `authority_id` | tstr ≤ 256 byte |
| `mcc_version` | uint8 |
| `serial_number` | UUID v4 raw, 16 byte |

**Campi raccomandati (non-critical):** `agent_role`, `capabilities`, `issuer_timestamp`, `policy_url`, `delegated_agent_pk`.

```cddl

2.5 Politica delle Chiavi: Separazione Obbligatoria
In ATP‑Full (§6.3), l'uso di chiavi separate è OBBLIGATORIO:

agent_pk (X25519, 32 byte): esclusivamente per ECDH nel TLS handshake

agent_sign_pk (Ed25519, 32 byte): esclusivamente per firme MCC e proof-of-possession

Un receiver ATP‑Full DEVE verificare che i 32 byte di agent_pk e agent_sign_pk siano distinti. MCC con agent_pk == agent_sign_pk → ERR_MISSING_CRITICAL_CLAIM.

⚠ AVVISO DI SICUREZZA: Il dual-use X25519/Ed25519 tramite conversione RFC 7748 è DEPRECATO in ATP-Core-Minimal e sarà rimosso in ATP v2.0. Viola la separazione dei ruoli crittografici. Se usato, includere "dual_use": true nella MCC.

3. Architettura di Trasporto
3.1 Stack di Trasporto
Versioni QUIC: QUIC v1 (RFC 9000) obbligatorio; QUIC v2 (RFC 9369) opzionale. La negoziazione avviene tramite RFC 8999 e DEVE completarsi prima di qualsiasi scambio ATP. Il TLS handshake DEVE completarsi prima che ATP invii qualsiasi frame.

Cipher suites TLS 1.3 (in ordine di preferenza):

Cipher Suite	Status
TLS_CHACHA20_POLY1305_SHA256	Raccomandata (performance)
TLS_AES_256_GCM_SHA384	Raccomandata (enterprise)
TLS_AES_128_GCM_SHA256	Accettabile (legacy)
Suite senza forward secrecy, RC4, 3DES, SHA-1, MD5	VIETATA
certificate_type: raw_public_key (RFC 7250), OID id-X25519 (RFC 8410).

ALPN: "atp/1" per tutta la famiglia major 1.x.

Vincolo normativo: la raw public key nel TLS handshake DEVE essere identica ad agent_pk nella MCC. Mismatch → ERR_IDENTITY_MISMATCH.

Formato versione: "MAJOR.MINOR.PATCH" con confronto numerico componente-per-componente.

3.2 NAT Traversal
Le implementazioni DOVREBBERO supportare un meccanismo di relay (es. TURN, RFC 8656). La specifica del relay è fuori scope.

3.3 Sequenza di Connessione Completa
Timeout di setup globale: se le fasi obbligatorie FASE 1–3 non si completano entro connection_setup_timeout_ms (default 10.000 ms), il peer DEVE chiudere la connessione QUIC.

FASE 1 — QUIC + TLS Handshake (OBBLIGATORIA)

Initiator invia QUIC Initial con ClientHello, raw_public_key = initiator_agent_pk, ALPN = "atp/1"

Responder risponde con ServerHello, raw_public_key = responder_agent_pk

QUIC stabilito, TLS handshake completato, stream aperti.

FASE 2 — Version Negotiation — stream 0 (OBBLIGATORIA)

Initiator → VERSION_PROPOSE (0x30): array versioni + parametri di sessione.

Responder → VERSION_ACK (0x31): versione scelta + parametri negoziati.

Se nessuna versione comune → ERR_ATP_VERSION_UNSUPPORTED (supported_versions obbligatorio).

FASE 3 — MCC Exchange & Identity Binding (OBBLIGATORIA)

Initiator → MCC_BIND_REQUEST: MCC + nonce_i (16 byte, CSPRNG).

Responder verifica MCC dell'initiator; risponde con MCC_BIND_RESPONSE: propria MCC + nonce_r + firma Ed25519 su nonce_i || "atp-bind-response"

Initiator verifica MCC del Responder e firma su nonce_i; risponde con MCC_BIND_CONFIRM: firma Ed25519 su nonce_r || "atp-bind-confirm"

Responder verifica la firma su nonce_r → binding completato. Il binding è bidirezionale: un binding unilaterale NON DEVE essere accettato.

FASE 4 — Capability Exchange — stream 0 (OPZIONALE)

Dopo la FASE 3 e prima dei task stream, i peer POSSONO scambiarsi CAPABILITY_EXCHANGE (0x50) su stream 0. Un peer che invia DEVE aspettare la risposta entro capability_exchange_timeout_ms = 2000 ms.

cddl
capability_exchange = {
  "header": frame_header,
  "capabilities": {* tstr => any},
  ? "required": [+ tstr]
}
Il campo "required" elenca le capability obbligatorie: se il receiver non le supporta → ERR_STREAM_PROTOCOL_VIOLATION + chiusura.

FASE 5 — Task Streams (OPERATIVA)

Apertura di stream pari (2, 4, 6...) da parte del client per l'invio dei task. Questa fase ha durata indefinita per tutta la sessione READY.

3.4 State Machine della Connessione
text
INIT
  ↓ (QUIC+TLS OK; initiator apre stream 0)
QUIC_OK
  ↓ (VERSION_PROPOSE/VERSION_ACK)
VERSION_NEGOTIATED
  ↓ (MCC_BIND_REQUEST → MCC_BIND_RESPONSE → MCC_BIND_CONFIRM)
MCC_BOUND
  ↓ (CAPABILITY_EXCHANGE completato / timeout / assente)
READY
  ↘ (CONTROL_SHUTDOWN)
CLOSING → CLOSED

Da qualunque stato (eccetto CLOSED): errore fatale → FAILED → CLOSED
3.5 Politica di Compatibilità e Meccanismo di Estensione
Versioni minor (1.5 → 1.6): compatibili; funzionalità aggiuntive via extensions.

Versioni major (1.x → 2.x): possibili breaking changes; richiedono nuovo handshake ALPN.

Formato: "MAJOR.MINOR.PATCH" con confronto numerico componente-per-componente.

VERSION_PROPOSE e VERSION_ACK POSSONO contenere: ? "extensions": {* tstr => any}. Namespace chiavi: "vendor.nomefirma.feature" o "core.feature". Un receiver che non riconosce una chiave DEVE ignorarla silenziosamente.

3.6 Politica 0-RTT
⚠ ATTENZIONE: 0-RTT è DISABILITATO per default. Il riutilizzo di ticket 0-RTT senza cache anti-replay persistente espone a replay attack.

Se 0-RTT viene abilitato:

I task 0-RTT DEVONO essere idempotenti (registro D.3).

Il server DEVE persistere la cache anti-replay su storage stabile; senza cache persistita NON DEVE abilitare 0-RTT.

Se cache non disponibile → ERR_STREAM_PROTOCOL_VIOLATION, disabilita 0-RTT per la sessione.

3.7 Discovery DNS
Record SRV + TXT con: endpoint QUIC, raw public key X25519, authority_id, versione ATP (formato "1.6.1"). DNSSEC è obbligatorio per ATP-Full; senza DNSSEC: conformità solo ad ATP-Core-Minimal.

5. Gestione delle Revoche (ATP-Revocation)
Modulo concettuale separato. Obbligatorio per ATP-Full; facoltativo per ATP-Core-Minimal.

5.1 Cuckoo Filters — Parametri e Formato Binario
Parametro	Valore Normativo
False positive rate	≤ 0,1%
bucket_size	4 slot
fingerprint_bits	16 bit
Load factor massimo	95%
Calcolo del fingerprint (interoperabilità obbligatoria):

text
preimage = serial_number || expiry_date_bytes   // expiry_date come uint64 BE (8 byte)
fingerprint = BLAKE3(preimage)[0:2]             // primi 2 byte (big-endian)
// Se fingerprint == 0x0000 → sostituire con 0x0001 (0x0000 riservato per slot vuoto)
Funzioni di lookup (interoperabilità obbligatoria):

text
h1(x) = BLAKE3(0x68 || x) mod bucket_count
// 0x68 = 'h', domain prefix per h1

h2(x) = (h1(x) XOR (BLAKE3(0x66 || fingerprint(x)) mod bucket_count)) mod bucket_count
// 0x66 = 'f', domain prefix per h2
// mod esterno garantisce h2 ∈ [0, bucket_count) anche per bucket_count non potenza di 2
Inserimento: cuckoo eviction (max 500 tentativi). Se eviction fallisce o load factor supera 95% → espandere il filtro: bucket_count raddoppiato, ricopiare entry, propagare via gossip con nuovo filter_id.

Formato binario CFT1:

Offset	Size	Campo
0	4 byte	Magic: 0x43465431 ("CFT1")
4	4 byte	bucket_count (uint32 BE)
8	2 byte	bucket_size (uint16 BE) = 4
10	2 byte	fingerprint_bits (uint16 BE) = 16
12	N byte	fingerprint_data: bucket_count × 4 slot × 2 byte

5.3 Gossip Epidemic (Push-Pull) — con Rate‑Limit e Dimensione Massima
Ogni peer accetta al massimo 1 snapshot ogni gossip_interval_s da un singolo sender.

Dimensione massima dello snapshot: max_snapshot_size = 1 MiB.

Gli snapshot devono essere incrementali (filter_id crescente); quelli con ID minore o uguale vengono ignorati.

Parametro	Valore Default	Range Configurabile
gossip_fanout	3 peer casuali per round	1–16
gossip_interval_s	5 secondi	1–60
urgent push	Immediato a tutti i peer attivi	—
Convergenza attesa	O(log N) round	—
Snapshot history: quando un peer riceve un nuovo snapshot con filter_id diverso, DEVE mantenere il precedente per ≥ 2 × gossip_interval_s. Un serial_number è revocato se presente in qualsiasi snapshot della history.

5.4 Grace Period e Fallback Policy (Rivista — Graduated Revocation)
python
age_s = (now_ms / 1000) - snapshot.timestamp

if age_s < grace_period_s (default 300):
    status = CONFIRMED      # fiducia piena
elif age_s < 2 * grace_period_s:
    status = STALE          # accettato con warning
else:
    status = UNCERTAIN      # incertezza

if status == UNCERTAIN:
    if critical_capabilities_present:
        fail‑closed (ERR_REVOKED)
    else:
        fail‑open (accetta, log REVOCATION_UNCERTAIN)
Se age_s ≥ grace_period_s: tentare OCSP-like (§5.5). Se OCSP fallisce e critical capabilities → fail-closed; se no critical → fail-open. Sempre: loggare REVOCATION_UNCERTAIN.

5.6 Root Store e Governance (Rivisto — Catena di Manifest)
Il Root Store è ora una catena di manifest firmati. Ogni nuovo manifest (root_store_manifest) contiene:

  "authority_id": tstr,
  "public_key": bstr .size 32,

Verifica di un nuovo manifest:

prev_manifest_id DEVE corrispondere all'ID del manifest attuale.

manifest_version > versione corrente.

Almeno quorum_threshold firme valide da autorità non revocate (revoked ≠ true).

Timestamp > timestamp del manifest corrente.

Se tutti i controlli passano, il nuovo manifest viene adottato.

Revoca autorità: una authority_entry con revoked: true non contribuisce al quorum. MCC già in uso non sono automaticamente invalidate (usare il Cuckoo Filter per revocarle).

Bootstrap Root Store: distribuito fuori banda (es. hash pinnato del manifest genesis). Implementazioni ATP-Full DEVONO documentare il processo di bootstrap.

6. Workflow, Anti-Replay e Rate Limiting
6.1 Anti-Replay — con Gestione Overflow Esplicita
Si applica ESCLUSIVAMENTE ai frame TASK_REQUEST. Procedura per ogni TASK_REQUEST ricevuto:

Verificare timestamp ∈ [now_ms − clock_skew_ms, now_ms + clock_skew_ms]. In caso contrario → TASK_ERROR ERR_CLOCK_SKEW, chiudi stream.

Verificare che task_id non sia in cache. Se presente → scartare silenziosamente (nessun ACK).

Inserire task_id in cache con TTL = anti_replay_ttl_ms.

Verificare rate limit (§6.2). Se in violation → TASK_ERROR ERR_RATE_LIMITED senza inviare TASK_ACK.

Gestione overflow cache: Se la cache è piena, il receiver DEVE rifiutare il task con ERR_RATE_LIMITED e retry_after_ms calcolato sul TTL residuo dell'entry più vecchia (sliding window). Loggare CACHE_OVERFLOW_DETECTED.

6.2 Rate Limiting
Algoritmo: token bucket per peer (identificato da agent_pk).

Capacità: 20 token. Ricarica: rate_limit_rps token/s (default 100 rps).

Violazione: → ERR_RATE_LIMITED.

Ban automatico: 10 violazioni in 60s → ban 300s. Identificato da agent_pk, persistente tra connessioni (best-effort al riavvio).

Durante ban: accettare binding ma rifiutare tutti i TASK_REQUEST con ERR_RATE_LIMITED e retry_after_ms = tempo residuo del ban.

7. Semantica degli Stream QUIC
7.1 Allocazione degli Stream ID
Stream 0: control stream — VERSION_PROPOSE/ACK, MCC_BIND_, CONTROL_, ERROR, ROOT_STORE_UPDATE, CAPABILITY_EXCHANGE.

Stream pari ≥ 2 (2, 4, 6...): task stream aperti dal client.

Stream dispari: riservati a usi futuri; NON DEVONO essere aperti in ATP v1.x.

7.2 Ciclo di Vita di un Task Stream
Flusso tipico:

text
Client apre stream pari
  → TASK_REQUEST
  ← TASK_ACK (entro ack_window_ms = 200ms)
  ← TASK_RESPONSE (partial=true, sequence=0)
  ← TASK_RESPONSE (partial=true, sequence=1)
  ← TASK_RESPONSE (partial=false, sequence=2) ← risposta finale
  ← Server FIN (entro stream_close_timeout_ms)
  → Client FIN
stream chiuso
Il server DEVE inviare FIN entro stream_close_timeout_ms dall'ultima TASK_RESPONSE.

Il client DEVE inviare FIN entro stream_close_timeout_ms dalla ricezione del FIN del server.

Il client NON deve inviare FIN prima del server (evitare half-close deadlock).

TASK_RESPONSE prima di TASK_ACK → ERR_STREAM_PROTOCOL_VIOLATION → chiudi connessione.

7.3 Chiusura del Control Stream
Procedura CONTROL_SHUTDOWN:

Initiator invia CONTROL_SHUTDOWN(is_echo: false)

Initiator entra in CONTROL_CLOSING: non accetta nuovi task; completa quelli esistenti.

Receiver completa i propri task in-flight, risponde con CONTROL_SHUTDOWN(is_echo: true), entra in CONTROL_CLOSING.

Initiator aspetta il CONTROL_SHUTDOWN(is_echo: true) per drain window = stream_close_timeout_ms

Se non arriva entro drain window: invia QUIC CONNECTION_CLOSE e chiude.

NOTA — Chiusura simultanea: Se entrambi i peer inviano CONTROL_SHUTDOWN(is_echo: false) simultaneamente, ciascuno DEVE rispondere con is_echo: true come se fosse il receiver. Non è un errore.

NOTA — Scadenza MCC in sessione: Il server DOVREBBE verificare periodicamente expiry_date > now. Se la MCC scade durante una sessione → CONTROL_SHUTDOWN(is_echo: false, reason: "mcc_expired") e chiusura ordinata. Task in-flight completati entro deadline_ms.

7.4 Priorità (RFC 9218)
Le implementazioni DOVREBBERO rispettare il campo priority_hint nel TASK_REQUEST secondo la semantica RFC 9218. La priorità è un suggerimento: il server può ignorarla in caso di saturazione.

7.5 Flow Control
initial_max_stream_data DEVE essere ≥ max_batch_bytes × 2

Sender NON DEVE inviare task_payload > max_batch_bytes → ERR_TASK_TOO_LARGE (§8.1)

In caso di esaurimento credito flow control: loggare FLOW_CONTROL_BLOCKED(stream_id), non resettare lo stream.

initial_max_data (connection-level) DEVE essere ≥ initial_max_stream_data × min(rate_limit_rps, 100)

8.2 Comportamento Prescrittivo
Chiusura connessione: errori 0x01–0x08 (se su stream 0) e 0x0E–0x7F. Inviare ERROR su stream 0, poi chiudere QUIC.

Chiusura stream: errori 0x09–0x0C, 0x0E (se su task stream). Inviare TASK_ERROR sul task stream, poi reset stream.

Recuperabili: 0x0D — rispettare retry_after_ms o backoff esponenziale (inizio 1s, fattore 2, max 60s). Per ERR_CLOCK_SKEW, il server DEVE includere server_time_ms; il client può riprovare con timestamp corretto.

9. Security Considerations
9.1 Modello di Minaccia
ATP è progettato per ambienti zero-trust. Attori di minaccia nel perimetro:

Peer malevoli: tentano impersonazione, riutilizzo credenziali scadute/revocate, task fraudolenti.

Attaccanti di rete: MITM passivo/attivo, replay, downgrade.

Autorità compromesse: emissione MCC fraudolente o necessità di rimozione dal Root Store.

Fuori scope: attacchi side-channel fisici, compromissione OS, DoS volumetrica a livello QUIC/IP.

9.2 Sicurezza del Binding Crittografico
Catena di trust:

text
Root Store → authority_pk → MCC.authority_sig → MCC.root_hash → agent_sign_pk → proof-of-possession
Garanzie:

Impossibile presentare MCC valida senza agent_sign_sk.

Raw TLS public key ≠ agent_pk → binding fallisce.

Alterare foglie invalida root_hash.

leaf_hash forgiato rilevato perché il receiver ricalcola (e il campo non viene trasmesso).

DIPENDENZA CRITICA: La sicurezza dipende dall'integrità del Root Store. Proteggere con quorum k-of-n (quorum_threshold ≥ 3). Distribuire il Root Store genesis fuori banda con hash pinnato. La catena di manifest permette aggiornamenti sicuri.

9.3 Separazione delle Chiavi Crittografiche
Il dual-use X25519/Ed25519 viola la separazione dei ruoli. In ATP-Full: chiavi separate obbligatorie (§2.5). In ATP-Core-Minimal: dual-use DEPRECATO (§2.5).

9.4 Anti-Replay e Clock Skew
Unicità task_id (UUID v4): probabilità collisione ≈ 2⁻⁶¹ per 10⁹ task_id.

Finestra temporale [now ± clock_skew_ms]: limita replay a 2 × clock_skew_ms (default 10s).

Ambienti ad alta sicurezza: clock_skew_ms ≤ 5.000 ms, NTP stratum ≤ 2.

Fallback clock skew: se il server riceve timestamp fuori finestra, risponde con server_time_ms nel TASK_ERROR; il client può riprovare.

9.5 Sicurezza dei Cuckoo Filter
False positive ≈ 0,1%: fail-closed è il trade-off corretto.

Forgery fingerprint: neutralizzato dalla firma Ed25519 sull'intero snapshot.

Rate limit sulle verifiche gossip: max 1 snapshot/s per peer mittente.

9.6 Sicurezza del Gossip
La firma Ed25519 sull'intero snapshot garantisce che solo snapshot firmati da autorità legittima vengano accettati.

Rate limit sulle verifiche gossip: max 1 snapshot/s per peer mittente (raccomandato) per mitigare flooding.

Dimensione massima snapshot: 1 MiB per prevenire DoS.

9.7 Sicurezza 0-RTT
0-RTT disabilitato per default. Se abilitato: cache anti-replay persistente obbligatoria; solo task_type idempotenti (registro D.3).

9.8 Confidenzialità e Forward Secrecy
Tutto il traffico ATP è cifrato con TLS 1.3 su QUIC.

Forward secrecy tramite ephemeral DH (X25519).

Le MCC non contengono informazioni riservate per design.

9.9 Resistenza al Downgrade
VERSION_PROPOSE viaggia dentro il tunnel TLS 1.3 cifrato e autenticato: modifica causerebbe violazione AEAD e chiusura QUIC.

ALPN "atp/1" previene downgrade a versioni major diverse a livello TLS.

9.10 Canali Laterali
Ed25519 usa firma deterministica (RFC 8032): elimina leakage nonce casuale presente in ECDSA.

Le implementazioni DOVREBBERO usare comparazioni constant-time per verifica firme Ed25519 e lookup Cuckoo Filter.

9.11 Delegation
Il campo non-critical delegated_agent_pk permette a un agente di agire per conto di un altro, abilitando scenari di orchestrazione e proxy. Il receiver DEVE verificare che il campo sia valido e può applicare politiche di autorizzazione.

## Abstract

The Agent Transport Protocol (ATP) defines a secure application-level protocol for the exchange of tasks and information between autonomous AI agents. ATP combines cryptographic identities based on Merkle-Claim Cards (MCC), mutually authenticated transport over QUIC + TLS 1.3 with raw public keys (RFC 7250), and a distributed revocation subsystem based on Cuckoo Filters synchronised via gossip epidemic propagation.

This document specifies version 1.6.1 of the protocol, which is backwards-compatible with all conformant ATP 1.5.x and 1.6.x implementations. This revision introduces several architectural improvements: a Root Store chain for secure updates, a graduated revocation policy (CONFIRMED/STALE/UNCERTAIN), mandatory key separation, removal of the ambiguous leaf_hash field, a clock-skew fallback mechanism, and delegation support.

## 1. Introduction

As AI systems evolve toward ecosystems of co-operating autonomous agents, a need arises for a purpose-built protocol that addresses three challenges simultaneously: (1) establishing verifiable agent identity without relying on a centralised Public Key Infrastructure, (2) providing mutually authenticated zero-trust transport, and (3) propagating revocation decisions rapidly across a potentially large, decentralised peer set.

Existing protocols solve subsets of these challenges. OAuth 2.0 and JWT address identity for human users but lack agent-native constructs such as capability declarations and cryptographic binding of identity to a transport key. X.509 PKI provides strong identity but is operationally centralised and was not designed for the dynamic lifecycle of AI agents. gRPC over mTLS offers mutual authentication but again depends on X.509 and provides no distributed revocation.

ATP addresses all three challenges in a single, coherent protocol designed from the ground up for agent workloads.

## 1.2. Terminology

**Agent:** An autonomous software entity that sends and receives ATP tasks.

**MCC:** Merkle-Claim Card. A self-contained cryptographic identity object encoding an agent's public keys, role, capabilities, and expiry.

**Authority:** An entity that signs MCC commitment objects, identified by an Ed25519 public key held in the local Root Store.

**Root Store:** A local list of trusted authority public keys, distributed as a signed Root Store Manifest chain.

**Cuckoo Filter:** A probabilistic data structure used to represent the set of revoked MCC serial numbers.

**Gossip:** An epidemic propagation protocol used to synchronise Cuckoo Filter snapshots across peers.

**Initiator:** The agent that opens the QUIC connection.

**Responder:** The agent that accepts the QUIC connection.

**ATP-Full:** The complete conformance profile including revocation subsystem, Root Store Manifest, and mandatory key separation.

**ATP-Core-Minimal:** A reduced conformance profile suitable for embedded or closed environments.

**CDDL:** Concise Data Definition Language [RFC8610], used to specify CBOR structures throughout this document.

---

## 2. Merkle-Claim Cards (MCC)

Each agent possesses an MCC: a Merkle Abstract Syntax Tree (MAST) constructed with BLAKE3, encoded deterministically in CBOR, and signed by an authority whose public key appears in the local Root Store. An MCC is self-contained; no online lookup is required to verify it beyond access to the Root Store and, for ATP-Full, the local Cuckoo Filter.

---

## 2.1. Merkle Tree Construction

Each leaf is computed as follows (all concatenations are byte-level):
L_i = BLAKE3(
0x00 || ; leaf domain prefix (REQUIRED)
Salt_128bit || ; 16 bytes, CSPRNG, unique per leaf
uint16_BE(len(Key_bytes)) || ; key length, big-endian
Key_bytes || ; key as UTF-8 bytes
uint32_BE(len(Val_bytes)) || ; value length, big-endian
Val_bytes ; value bytes
)

text

Leaf ordering: keys MUST be sorted by byte-wise lexicographic comparison of their UTF-8 representation. A shorter key that is a prefix of a longer key precedes it. The leaf_index is zero-based.

Tree construction rules:
- The tree MUST be a complete binary tree.
- If N is not a power of two, the last level MUST be padded by duplicating the last leaf (see RFC 6962, Section 2.1).
- Internal nodes: Node = BLAKE3(0x01 || Left || Right). The 0x01 prefix is REQUIRED to prevent second-preimage attacks. Implementations that omit this prefix are NOT conformant with ATP and are NOT interoperable.
- Special case -- single-leaf tree (N=1): root_hash = L_0. No additional hashing is applied. A single-leaf tree has no internal node.

---

## 2.2. Authority Signature and MCC Verification

The authority signs a deterministically encoded CBOR commitment object. The signed object contains exactly five fields and MUST NOT include the 'leaves' array.

Signed commitment (key order per RFC 8949, Section 4.2.1, sorted by key byte length then lexicographically):
{
"root_hash": bstr .size 32, ; 9-byte key
"expiry_date": uint, ; 11-byte key
"mcc_version": uint, ; 11-byte key ("e" < "m")
"authority_id": tstr, ; 12-byte key
"serial_number": bstr .size 16 ; 13-byte key
}

authority_sig = Ed25519Sign(authority_sk, CBOR_canonical(commitment))

text

The normative MCC verification procedure consists of eight steps that MUST be performed in the order given. Failure at any step MUST cause the receiver to abort the binding with the indicated error code.

1. Verify mcc_version is supported; otherwise: ERR_ATP_VERSION_UNSUPPORTED.
2. Verify expiry_date > floor(now_ms / 1000); otherwise: ERR_REVOKED.
3. Verify all fields listed in critical_mask exist among the leaves. An unrecognised critical_mask entry MUST cause ERR_MISSING_CRITICAL_CLAIM (fail-closed, analogous to RFC 5280, Section 4.2).
4. Recompute every leaf_hash from (key, value, salt) using the formula in Section 2.1. The receiver MUST NOT rely on any leaf_hash field in the wire format (it has been removed in v1.6.1). Recompute root_hash and verify it matches mcc.root_hash; otherwise: ERR_INVALID_ROOT.
5. Retrieve authority_pk from the local Root Store. If absent or revoked: ERR_INVALID_ROOT.
6. Verify authority_sig = Ed25519Verify(authority_pk, CBOR_canonical(commitment)); otherwise: ERR_BAD_SIGNATURE.
7. Verify the agent_pk leaf value matches the raw TLS public key used in the QUIC handshake; otherwise: ERR_IDENTITY_MISMATCH.
8. (ATP-Full only) Verify serial_number is absent from the local Cuckoo Filter; otherwise: ERR_REVOKED. In ATP-Core-Minimal this step is OPTIONAL.

> NOTE: The leaf_hash field in the wire format has been REMOVED in ATP v1.6.1. A receiver MUST recompute it from key/value/salt. A forger who modifies key/value/salt is detected at step 4.

## 2.3. Critical Mask

The following claims MUST appear in critical_mask for every conformant MCC:

| Field | Type and Constraints |
|-------|----------------------|
| agent_pk | X25519 public key, 32 bytes |
| agent_sign_pk | Ed25519 public key, 32 bytes (Sec. 2.5) |
| expiry_date | uint64, Unix epoch seconds |
| authority_id | tstr, <= 256 bytes |
| mcc_version | uint8 |
| serial_number | UUID v4, raw 16 bytes |

The following claims are RECOMMENDED but non-critical: agent_role, capabilities, issuer_timestamp, policy_url, delegated_agent_pk.

2.5. Key Separation Policy
In ATP-Full (Section 6.3), the use of separate keys is REQUIRED:

agent_pk (X25519, 32 bytes): MUST be used exclusively for ECDH key exchange in the TLS handshake.

agent_sign_pk (Ed25519, 32 bytes): MUST be used exclusively for MCC signatures and proof-of-possession.

An ATP-Full receiver MUST verify that the 32-byte values of agent_pk and agent_sign_pk are distinct. An MCC where agent_pk == agent_sign_pk MUST be rejected with ERR_MISSING_CRITICAL_CLAIM.

WARNING: Dual-use of X25519/Ed25519 via RFC 7748 conversion is DEPRECATED in ATP-Core-Minimal and will be removed in ATP v2.0. It violates cryptographic role separation. If used, the MCC MUST include "dual_use": true.

3. Transport Architecture
3.1. Transport Stack
QUIC versions: QUIC v1 [RFC9000] is REQUIRED; QUIC v2 [RFC9369] is OPTIONAL. Version negotiation follows [RFC8999] and MUST complete before ATP transmits any frame. The TLS 1.3 handshake MUST complete before ATP sends any frame.

TLS 1.3 cipher suites, in preference order:

Cipher Suite	Status
TLS_CHACHA20_POLY1305_SHA256	RECOMMENDED
TLS_AES_256_GCM_SHA384	RECOMMENDED
TLS_AES_128_GCM_SHA256	acceptable (legacy)
Any suite without forward secrecy, RC4, 3DES, SHA-1, MD5	MUST NOT be used
certificate_type: raw_public_key [RFC7250], OID id-X25519 [RFC8410].

Application-Layer Protocol Negotiation (ALPN): "atp/1" for all minor versions within the 1.x major family.

The raw public key used in the TLS handshake MUST be identical to the agent_pk leaf in the MCC. A mismatch MUST result in ERR_IDENTITY_MISMATCH.

Version strings use the format "MAJOR.MINOR.PATCH" with numeric component-wise comparison.

3.2. NAT Traversal
Implementations SHOULD support a relay mechanism such as TURN [RFC8656]. The relay specification is outside the scope of this document.

3.3. Connection Sequence
A connection is established in five phases. Phases 1 through 3 are REQUIRED; Phase 4 is OPTIONAL; Phase 5 is the operational phase.

Global setup timeout: if Phases 1-3 do not complete within connection_setup_timeout_ms (default 10,000 ms, a local parameter that is not negotiated), the peer MUST close the QUIC connection.

Phase 1 -- QUIC + TLS Handshake (REQUIRED)

Initiator sends QUIC Initial with ClientHello, raw_public_key = initiator_agent_pk, ALPN = "atp/1".

Responder replies with ServerHello, raw_public_key = responder_agent_pk.

QUIC is established; TLS handshake completes; streams are open.

Phase 2 -- Version Negotiation on stream 0 (REQUIRED)

Initiator -> VERSION_PROPOSE (0x30): array of supported versions plus session parameters.

Responder -> VERSION_ACK (0x31): chosen version plus negotiated parameters.

If no common version exists -> ERR_ATP_VERSION_UNSUPPORTED (supported_versions MUST be included).

Phase 3 -- MCC Exchange and Identity Binding (REQUIRED)

Initiator -> MCC_BIND_REQUEST: MCC + nonce_i (16 bytes, CSPRNG).

Responder verifies the Initiator's MCC (eight-step procedure, Section 2.2); replies with MCC_BIND_RESPONSE: own MCC + nonce_r + Ed25519 signature over (nonce_i || "atp-bind-response").

Initiator verifies Responder's MCC and the signature over nonce_i; replies with MCC_BIND_CONFIRM: Ed25519 signature over (nonce_r || "atp-bind-confirm").

Responder verifies the signature over nonce_r. Binding is complete. Unilateral binding MUST NOT be accepted.

Phase 4 -- Capability Exchange on stream 0 (OPTIONAL)

After Phase 3 and before task streams, peers MAY exchange CAPABILITY_EXCHANGE (0x50) on stream 0. A sender MUST wait for a response or until capability_exchange_timeout_ms (2,000 ms) elapses. The 'required' field lists mandatory capabilities; if the receiver does not support them, it MUST respond with ERR_STREAM_PROTOCOL_VIOLATION and close the connection.

Phase 5 -- Task Streams (operational, indefinite duration)

The client opens QUIC streams with even stream IDs (2, 4, 6, ...) to send tasks. This phase continues for the lifetime of the connection.

3.4. Connection State Machine
text
INIT
  | QUIC + TLS handshake complete; Initiator opens stream 0
  v
QUIC_OK
  | VERSION_PROPOSE / VERSION_ACK
  v
VERSION_NEGOTIATED
  | MCC_BIND_REQUEST / MCC_BIND_RESPONSE / MCC_BIND_CONFIRM
  v
MCC_BOUND
  | CAPABILITY_EXCHANGE complete / timeout / skipped
  v
READY <------ normal task operation
  | CONTROL_SHUTDOWN(is_echo: false)
  v
CLOSING
  | CONTROL_SHUTDOWN(is_echo: true) + FIN
  v
CLOSED

From any state except CLOSED:
  fatal error --> FAILED --> CLOSED
3.5. Compatibility and Extension Mechanism
Minor versions (e.g., 1.5 -> 1.6): backwards-compatible. New functionality is introduced via extensions.

Major versions (e.g., 1.x -> 2.x): may introduce breaking changes and require a new ALPN token.

VERSION_PROPOSE and VERSION_ACK MAY contain an optional "extensions" map ({* tstr => any}). Key namespaces: "vendor.<name>.<feature>" for proprietary extensions and "core.<feature>" for future ATP standardised extensions. A receiver that does not recognise an extension key MUST silently ignore it.

3.6. 0-RTT Policy
0-RTT is DISABLED by default. If enabled:

0-RTT tasks MUST be idempotent (see Appendix D.3).

The server MUST persist the anti-replay cache to stable storage. Without a persistent cache, the server MUST NOT enable 0-RTT.

If the cache is unavailable at connection time, the server MUST reject 0-RTT with ERR_STREAM_PROTOCOL_VIOLATION and disable 0-RTT for the session.

3.7. DNS Discovery
An ATP endpoint MAY publish itself via DNS SRV and TXT records. The TXT record MUST include the QUIC endpoint, the raw X25519 public key, the authority_id, and the ATP version string (e.g., "1.6.1"). DNSSEC [RFC4033] is REQUIRED for ATP-Full; without DNSSEC, a deployment can claim only ATP-Core-Minimal conformance.

4. Frame Wire Format
All ATP frames MUST be serialised in deterministic CBOR [RFC8949, Section 4.2]. Map keys MUST be tstr (text strings). A map using uint keys MUST be rejected with ERR_MISSING_CRITICAL_CLAIM.

4.1. Frame Header
cddl
frame_header = {
  "frame_type":  uint,
  "frame_id":    bstr .size 16,  ; UUID v4, frame deduplication
  "task_id":     bstr .size 16,  ; nil UUID (0x00 * 16) for control
  "timestamp":   uint,           ; Unix epoch milliseconds (uint64)
  "atp_version": tstr            ; version negotiated in the session
}
task_id MUST be the nil UUID for: VERSION_PROPOSE/ACK, MCC_BIND_, CONTROL_, ERROR, ROOT_STORE_UPDATE, and CAPABILITY_EXCHANGE frames. The anti-replay cache applies EXCLUSIVELY to TASK_REQUEST frames.

0x10	CONTROL_SHUTDOWN	0	bidirectional

4.3. TASK_REQUEST
cddl
task_request = {
  "header":          frame_header,
  "task_type":       tstr .size (1..128),
  "task_payload":    bstr,
  "deadline_ms":     uint,
  ? "metadata":      {* tstr => any},
  ? "priority_hint": uint .le 7
}
task_payload MUST be <= max_batch_bytes. A larger payload MUST be rejected with ERR_TASK_TOO_LARGE.

deadline_ms: the timer starts from receipt of the complete TASK_REQUEST. The server MUST send TASK_ACK within ack_window_ms (200 ms, non-negotiable) and MUST complete processing within deadline_ms.

priority_hint: 0 (lowest) to 7 (highest). The server SHOULD respect it but MAY ignore it under load.

If partial = true, the sequence field is REQUIRED. The receiver MUST reorder by ascending sequence. A duplicate sequence number MUST cause ERR_STREAM_PROTOCOL_VIOLATION.

4.8. Task Stream State Machine
text
CREATED
  | TASK_REQUEST sent
  v
REQUEST_SENT
  |                     \
  | TASK_ACK received    \--(ack timeout)--> FAILED
  v
ACKED
  |\
  | \-- TASK_RESPONSE (partial=false) --> COMPLETED
  |  \- TASK_ERROR -----------------> FAILED
  |
  | TASK_RESPONSE (partial=true)
  v
IN_PROGRESS
  |\
  | \-- TASK_RESPONSE (final) -------> COMPLETED
  |  \- TASK_ERROR / RESET_STREAM --> FAILED
TASK_ACK is a receipt acknowledgement, not a completion acknowledgement. The sender MUST NOT automatically retransmit failed tasks.

5. Revocation Subsystem
The revocation subsystem is REQUIRED for ATP-Full and OPTIONAL for ATP-Core-Minimal.

5.1. Cuckoo Filter Parameters and Binary Format
The following parameters are normative for interoperability:

False positive rate: <= 0.1 %

bucket_size: 4 slots

fingerprint_bits: 16 bits

Maximum load factor: 95 %

Fingerprint derivation (normative):

text
preimage    = serial_number || expiry_date_bytes
              ; expiry_date as uint64 big-endian (8 bytes)
fingerprint = BLAKE3(preimage)[0:2]
              ; first 2 bytes, big-endian
; If fingerprint == 0x0000, substitute 0x0001
; (0x0000 is reserved to denote an empty slot)
Lookup hash functions (normative):

text
h1(x) = BLAKE3(0x68 || x) mod bucket_count
         ; 0x68 = 'h', domain separation prefix
h2(x) = (h1(x) XOR (BLAKE3(0x66 || fp(x)) mod bucket_count))
          mod bucket_count
         ; 0x66 = 'f'; outer mod ensures h2 in [0, bucket_count)
Insertion uses cuckoo eviction with a maximum of 500 attempts. If eviction fails or the load factor exceeds 95%, the filter MUST be expanded: bucket_count is doubled, entries are re-inserted, and the new snapshot is propagated via gossip.

Binary format (CFT1):

Offset	Size	Field
0	4 B	Magic: 0x43465431 ("CFT1")
4	4 B	bucket_count (uint32 big-endian)
8	2 B	bucket_size (uint16 big-endian) = 4
10	2 B	fingerprint_bits (uint16 BE) = 16
12	N B	fingerprint_data: count4 slots2 bytes
5.2. Snapshot CDDL Schema
cddl
cuckoo_filter_snapshot = {
  "filter_id":    bstr .size 16,
  "timestamp":    uint,           ; Unix epoch seconds
  "authority_id": tstr,
  "bucket_count": uint,
  "entry_count":  uint,
  "buckets":      bstr,           ; CFT1 data starting at offset 12
  "signature":    bstr .size 64,
  ? "urgent":     bool
}

; Signature: Ed25519Sign(authority_sk,
;            CBOR_canonical(snapshot excluding "signature"))
; Receiver MUST verify signature and that entry_count matches
; the number of non-empty slots in the filter.
5.3. Gossip Epidemic (Push-Pull) — with Rate‑Limit and Maximum Size
A peer MUST accept at most 1 snapshot per gossip_interval_s from a given sender.

Snapshot size MUST NOT exceed max_snapshot_size (default 1 MiB).

Snapshots are incremental; a snapshot with a filter_id <= current is ignored.

Parameter	Default	Configurable
gossip_fanout	3 random peers	1 - 16
gossip_interval_s	5 seconds	1 - 60
urgent push	immediate	--
convergence (O)	O(log N)	--
Snapshot history: when a peer receives a new snapshot with a different filter_id, it MUST retain the previous snapshot for at least 2 * gossip_interval_s. A serial_number is considered revoked if it appears in any snapshot in the history.

5.4. Grace Period and Fallback Policy (Graduated Revocation)
python
age_s = floor(now_ms / 1000) - snapshot.timestamp

if age_s < grace_period_s (default 300):
    status = CONFIRMED      # full trust
elif age_s < 2 * grace_period_s:
    status = STALE          # accepted with warning
else:
    status = UNCERTAIN      # uncertainty

if status == UNCERTAIN:
    if critical_capabilities_present:
        fail‑closed (ERR_REVOKED)
    else:
        fail‑open (accept, log REVOCATION_UNCERTAIN)
If age_s >= grace_period_s: attempt OCSP-like query (Section 5.5). If OCSP fails AND critical capabilities present: fail-closed; if OCSP fails AND no critical capabilities: fail-open. Always: log REVOCATION_UNCERTAIN.

5.5. OCSP-like Fallback
cddl
ocsp_request = {

  "requester_pk":  bstr .size 32,
  "nonce":         bstr .size 16
}

ocsp_response = {
  "serial_number": bstr .size 16,
  "nonce":         bstr .size 16,
  "status":        &(good:0, revoked:1, unknown:2),
  "this_update":   uint,
  "next_update":   uint,

}

; Transport: HTTPS (TLS 1.3) to ocsp_url in authority_entry.
; Signature guarantees authenticity independently of the TLS chain.
5.6. Root Store and Governance (Revised — Manifest Chain)
The Root Store is now a chain of signed manifests. Each manifest includes a "prev_manifest_id" field to form a lightweight blockchain.

cddl
root_store_manifest = {
  "manifest_version": uint,
  "manifest_id":      bstr .size 16,
  "prev_manifest_id": bstr .size 16,   ; nil UUID for genesis
  "timestamp":        uint,
  "quorum_threshold": uint,            ; MUST be >= 3
  "authorities":      [+ authority_entry],
  "signatures":       [+ manifest_signature],
  ? "gossip_params":  {
      "gossip_fanout":     uint .ge 1 .le 16,
      "gossip_interval_s": uint .ge 1 .le 60
    }
}

authority_entry = {
  "authority_id":      tstr,
  "public_key":        bstr .size 32,
  "valid_from":        uint,
  ? "valid_until":     uint,
  ? "revoked":         bool,
  "ocsp_url":          tstr,
  ? "gossip_endpoint": tstr
}
A new manifest is accepted only if:

prev_manifest_id matches the current manifest ID,

manifest_version > current version,

at least quorum_threshold valid signatures from non‑revoked authorities (revoked != true),

timestamp > current timestamp.

Authority revocation: an authority_entry with revoked: true does not contribute to the quorum. MCCs already in use are NOT automatically invalidated; use the Cuckoo Filter to revoke them individually.

Root Store bootstrap: distributed out-of-band (e.g., a pinned hash of the genesis manifest). ATP-Full implementations MUST document their bootstrap process in their Security Considerations.

6. Workflow, Anti-Replay, and Rate Limiting
6.1. Anti-Replay — with Explicit Overflow Handling
Anti-replay applies EXCLUSIVELY to TASK_REQUEST frames. For each received TASK_REQUEST, the receiver MUST perform the following steps in order:

Verify timestamp is within [now_ms - clock_skew_ms, now_ms + clock_skew_ms]. If not: TASK_ERROR ERR_CLOCK_SKEW, close stream.

Verify task_id is absent from the cache. If present: silently discard (no ACK).

Insert task_id into the cache with TTL = anti_replay_ttl_ms.

Verify rate limit (Section 6.2). If in violation: TASK_ERROR ERR_RATE_LIMITED without sending TASK_ACK.

Send TASK_ACK.

Process the task.

Cache overflow management: On cache overflow, the receiver MUST respond with ERR_RATE_LIMITED and retry_after_ms = remaining TTL of the oldest entry (sliding window). Log CACHE_OVERFLOW_DETECTED. The receiver MUST NEVER silently accept a task without inserting its task_id into the cache.

6.2. Rate Limiting
Algorithm: token bucket per peer, keyed by agent_pk.

Capacity: 20 tokens. Refill rate: rate_limit_rps tokens/s (default 100 rps).

Violation: ERR_RATE_LIMITED.

Auto-ban: 10 violations within 60 s triggers a 300 s ban. Ban state is identified by agent_pk and SHOULD persist across reconnections (best-effort on restart).

During ban: accept the binding phase but reject all TASK_REQUESTs with ERR_RATE_LIMITED and retry_after_ms = remaining ban time.

7. QUIC Stream Semantics
7.1. Stream ID Allocation
Stream 0: control stream -- VERSION_PROPOSE/ACK, MCC_BIND_, CONTROL_, ERROR, ROOT_STORE_UPDATE, CAPABILITY_EXCHANGE.

Even stream IDs >= 2 (2, 4, 6, ...): task streams opened by the client.

Odd stream IDs: reserved for future use; MUST NOT be opened in ATP v1.x.

7.2. Task Stream Lifecycle
Typical flow:

text
Client opens even stream
  -> TASK_REQUEST
  <- TASK_ACK           (within ack_window_ms = 200 ms)
  <- TASK_RESPONSE      (partial=true, sequence=0)
  <- TASK_RESPONSE      (partial=true, sequence=1)
  <- TASK_RESPONSE      (partial=false, sequence=2) [final]
  <- Server FIN         (within stream_close_timeout_ms)
  -> Client FIN
  [stream closed]
The server MUST send FIN within stream_close_timeout_ms of the last TASK_RESPONSE.

The client MUST send FIN within stream_close_timeout_ms of receiving the server FIN.

The client MUST NOT send FIN before the server (avoid half-close deadlock).

A TASK_RESPONSE received before TASK_ACK MUST cause ERR_STREAM_PROTOCOL_VIOLATION and connection closure.

7.3. Control Stream Shutdown
Orderly shutdown procedure:

Initiator sends CONTROL_SHUTDOWN(is_echo: false).

Initiator enters CONTROL_CLOSING: no new tasks accepted; in-flight tasks continue.

Receiver completes its in-flight tasks, replies with CONTROL_SHUTDOWN(is_echo: true), enters CONTROL_CLOSING.

Initiator waits for CONTROL_SHUTDOWN(is_echo: true) up to drain_window = stream_close_timeout_ms.

If no reply within drain_window: send QUIC CONNECTION_CLOSE.

Upon receipt: both peers send FIN on stream 0.

NOTE: Simultaneous shutdown: if both peers send CONTROL_SHUTDOWN(is_echo: false) simultaneously, each MUST respond with is_echo: true as if it were the receiver. This is not an error.

NOTE: MCC expiry during session: the server SHOULD periodically verify expiry_date > now. If the MCC expires: CONTROL_SHUTDOWN(is_echo: false, reason: "mcc_expired") and orderly closure. In-flight tasks are completed within deadline_ms.

7.4. Priority (RFC 9218)
Implementations SHOULD honour the priority_hint field in TASK_REQUEST according to the semantics of [RFC9218]. Priority is advisory; a server MAY ignore it under saturation.

7.5. Flow Control
initial_max_stream_data MUST be >= max_batch_bytes * 2.

A sender MUST NOT send task_payload > max_batch_bytes; doing so MUST be rejected with ERR_TASK_TOO_LARGE.

On exhaustion of flow-control credit: log FLOW_CONTROL_BLOCKED(stream_id); do NOT reset the stream.

initial_max_data (connection-level) MUST be >= initial_max_stream_data * min(rate_limit_rps, 100).

8.2. Prescriptive Behavior
Connection closure (send ERROR on stream 0, then close QUIC): codes 0x01-0x08 (when on stream 0) and 0x0E-0x7F.

Stream closure (send TASK_ERROR on the affected task stream, then reset): codes 0x09-0x0C, 0x0E (on task streams).

Recoverable: ERR_RATE_LIMITED -- respect retry_after_ms or apply exponential backoff (initial 1 s, factor 2, max 60 s). For ERR_CLOCK_SKEW, the server MUST include server_time_ms in the TASK_ERROR frame; the client MAY retry with the corrected timestamp.

9. Security Considerations
9.1. Threat Model
ATP is designed for zero-trust environments. In-scope threats:

Malicious peers: impersonation, replay of expired or revoked credentials, fraudulent task injection.

Network attackers: passive and active MITM, replay, version downgrade.

Compromised authorities: issuance of fraudulent MCCs or the need to remove an authority from the Root Store.

Out of scope: physical side-channel attacks, OS compromise, volumetric DoS at the QUIC/IP layer.

9.2. Cryptographic Binding
Trust chain:

text
Root Store -> authority_pk -> MCC.authority_sig -> MCC.root_hash -> agent_sign_pk -> proof-of-possession
Security guarantees:

It is computationally infeasible to present a valid MCC without possession of agent_sign_sk.

A TLS raw public key that differs from agent_pk causes binding failure.

Modifying any leaf invalidates root_hash.

A forged leaf_hash is detected because the receiver recomputes it (and the field is not transmitted).

CRITICAL: Security depends on the integrity of the Root Store. Protect it with a k-of-n quorum (quorum_threshold >= 3). Distribute the genesis Root Store out-of-band with a pinned hash. The manifest chain allows secure updates.

9.3. Key Separation
Dual-use of X25519/Ed25519 violates cryptographic role separation. In ATP-Full, separate keys are REQUIRED (Section 2.5). In ATP-Core-Minimal, dual-use is DEPRECATED (Section 2.5).

9.4. Anti-Replay and Clock Skew
UUID v4 uniqueness: collision probability ~ 2^-61 per 10^9 task IDs.

Temporal window [now +/- clock_skew_ms]: limits replay to 2 * clock_skew_ms (default 10 s).

High-security environments: clock_skew_ms <= 5,000 ms; NTP stratum <= 2.

Clock-skew fallback: if the server receives a timestamp outside the window, it responds with server_time_ms in TASK_ERROR; the client can retry.

9.5. Cuckoo Filter Security
False-positive rate ~ 0.1%: fail-closed is the correct trade-off.

Fingerprint forgery is neutralised by the Ed25519 signature covering the entire snapshot.

Rate-limit gossip verification: RECOMMENDED maximum of 1 snapshot per second per sender.

9.6. Gossip Security
The Ed25519 signature over the entire snapshot guarantees that only snapshots from a legitimate authority are accepted.

Rate-limit gossip verification: RECOMMENDED maximum of 1 snapshot per second per sender to mitigate flooding.

Maximum snapshot size: 1 MiB to prevent DoS.

9.7. 0-RTT Security
0-RTT is disabled by default. When enabled, a persistent anti-replay cache is REQUIRED and only idempotent task types MUST be used (Appendix D.3).

9.8. Confidentiality and Forward Secrecy
All ATP traffic is encrypted with TLS 1.3 over QUIC.

Forward secrecy via ephemeral Diffie-Hellman (X25519).

MCCs contain no confidential information by design.

9.9. Downgrade Resistance
VERSION_PROPOSE travels inside the encrypted, authenticated TLS 1.3 tunnel; tampering causes AEAD failure and QUIC closure.

ALPN "atp/1" prevents downgrade to a different major version at the TLS layer.

9.10. Side Channels
Ed25519 uses deterministic signing (RFC 8032), eliminating the nonce leakage present in ECDSA.

Implementations SHOULD use constant-time comparison for Ed25519 signature verification and Cuckoo Filter lookups.

9.11. Delegation
The non-critical "delegated_agent_pk" field allows an agent to act on behalf of another, enabling orchestration and proxying scenarios. The receiver MUST verify that the field is valid and MAY apply authorisation policies.

Appendix E. Interoperability Guidelines
E.1. Minimum Core Tests (ATP-Core-Minimal)
Full handshake: QUIC+TLS, VERSION_PROPOSE/ACK, MCC_BIND_REQUEST/RESPONSE/CONFIRM.

Valid MCC: correct root_hash, valid authority signature, binding complete.

MCC with missing critical claim -> ERR_MISSING_CRITICAL_CLAIM.

MCC with unrecognised critical_mask entry -> ERR_MISSING_CRITICAL_CLAIM (fail-closed).

MCC with agent_pk != TLS raw public key -> ERR_IDENTITY_MISMATCH.

MCC without leaf_hash field (v1.6.1): binding MUST succeed (receiver recomputes).

Simple task: REQUEST -> ACK (within 200 ms) -> RESPONSE -> FIN.

Partial response: reorder by sequence; final response has max sequence.

Anti-replay: duplicate task_id silently discarded (no ACK).

Rate limiting: 10+ violations cause ban; ERR_RATE_LIMITED MUST include retry_after_ms.

CONTROL_SHUTDOWN: is_echo:false + is_echo:true response + drain window.

Simultaneous CONTROL_SHUTDOWN: each peer responds is_echo:true.

CAPABILITY_EXCHANGE: unrecognised field ignored; unsupported 'required' field -> connection closure.

MCC expiry during session: CONTROL_SHUTDOWN with reason "mcc_expired".

atp_version in all frames MUST match the version negotiated in VERSION_ACK.

E.2. Additional Full Tests (ATP-Full)
Cuckoo Filter: Appendix B vectors -> fingerprint 0x1721 in bucket 2 -> positive lookup.

Non-revoked serial_number -> negative lookup.

Alternate bucket formula: h2 distinct from h1 (test vector: h1=2, h2=1).

Gossip snapshot exchange: core.gossip.push / core.gossip.pull.

In-session MCC revocation: CONTROL_REVOKE_NOTIFY -> REVOKING -> in-flight tasks complete -> CONTROL_SHUTDOWN.

Root Store Manifest: quorum >= quorum_threshold -> accepted; quorum < threshold -> rejected.

ROOT_STORE_UPDATE: manifest_version > local -> adopted; <= local -> ignored.

Dual-use rejection (ATP-Full): MCC with agent_pk == agent_sign_pk -> ERR_MISSING_CRITICAL_CLAIM.

Zero fingerprint: BLAKE3(...)[0:2] == 0x0000 -> substitute 0x0001.

Snapshot history: during filter replacement, positive lookup on both the current and previous filter.

Root Store Chain: verify prev_manifest_id and quorum during update.

Clock skew fallback: ERR_CLOCK_SKEW with server_time_ms; client retries with corrected timestamp.

Graduated revocation: UNCERTAIN state with critical capabilities -> fail-closed; without -> fail-open.

Delegation: delegated_agent_pk field present and valid.
